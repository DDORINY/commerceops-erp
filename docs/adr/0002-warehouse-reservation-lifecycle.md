# ADR-0002: 창고 재고 예약과 출고 시점

- 상태: 승인
- 날짜: 2026-07-04

## 배경

결제와 실제 배송 사이에는 시간이 있다. 결제 시 창고 실재고를 바로 차감하면 아직 창고에 있는 상품과 시스템 수량이 어긋나고, 배송 시점까지 차감하지 않으면 다른 주문이 같은 수량을 사용할 수 있다.

## 결정

- 결제 완료 시 상품 가용 총재고를 차감하고 창고 재고에는 예약 수량을 설정한다.
- 송장 등록 또는 주문의 `SHIPPING` 전환 시 창고 실재고와 예약 수량을 함께 차감한다.
- 반품 승인 시 예약 원장이 기록한 원출고 창고로 실재고를 복구한다.
- `stock_reservations`의 `RESERVED → SHIPPED → RETURNED` 상태로 중복 실행을 방지한다.

## 결과

- 창고 실재고, 예약, 가용수량을 구분할 수 있다.
- 한 주문을 여러 창고에 나눠 할당할 수 있다.
- 주문 취소 시 예약 해제 흐름이 추가로 필요하다.
- 향후 피킹/패킹 상태를 예약과 출고 사이에 확장할 수 있다.
