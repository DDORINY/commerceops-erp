# ADR-0001: 다중 창고 도입기의 재고 기준값

- 상태: 승인
- 날짜: 2026-07-04

## 배경

기존 시스템은 `products.stock_quantity` 하나로 주문과 재고를 처리한다. 다중 창고를 한 번에 기준 시스템으로 바꾸면 주문·결제·반품·입고 로직을 동시에 수정해야 해 회귀 위험이 크다.

## 고려한 선택지

1. 즉시 창고 재고 합계를 유일한 기준으로 전환 — 최종 구조는 명확하지만 기존 핵심 흐름의 변경 폭이 크다.
2. 상품 총재고를 유지하고 창고 재고를 단계적으로 배정 — 과도기 정합성 규칙이 필요하지만 안전하게 전환할 수 있다.

## 결정

v0.6 단계 1에서는 `products.stock_quantity`를 전체 재고 기준으로 유지한다. 창고별 합계는 이 값을 초과할 수 없으며 창고 이동은 총재고를 변경하지 않는다.

## 결과

- 기존 주문·결제·반품 흐름을 깨지 않고 창고 기능을 도입할 수 있다.
- 미배정 재고가 존재할 수 있으므로 화면에 배정 수량과 미배정 수량을 구분해야 한다.
- 다음 단계에서 입고·예약·출고를 창고 단위로 전환한 뒤 `warehouse_stocks`를 기준 시스템으로 승격하는 후속 ADR이 필요하다.
