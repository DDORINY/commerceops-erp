# Architecture Decision Records

ADR은 되돌리기 어렵거나 여러 기능에 영향을 주는 기술 결정을 짧고 명확하게 기록하는 문서다.

## 파일명 규칙

```text
0001-order-shipment-state-boundary.md
0002-database-migration-tool.md
```

- 번호는 순차 증가한다.
- 제목은 결정 주제를 소문자 kebab-case로 작성한다.
- 한 번 승인한 ADR은 삭제하지 않고, 대체 결정이 생기면 새 ADR에서 이전 결정을 대체한다고 기록한다.

## 템플릿

```markdown
# ADR-0000: 결정 제목

- 상태: 제안 / 승인 / 대체 / 폐기
- 날짜: YYYY-MM-DD
- 결정자:
- 관련 문서:

## 배경

어떤 문제와 제약이 있는가?

## 고려한 선택지

1. 선택지 A: 장점/단점
2. 선택지 B: 장점/단점

## 결정

무엇을 선택했는가?

## 이유

왜 이 선택이 현재 조건에 가장 적합한가?

## 결과

얻는 것, 감수하는 비용, 후속 작업은 무엇인가?
```

## 우선 작성 권장 ADR

1. 주문과 배송 상태의 소유권 및 동기화 방식.
2. 결제, 재고, 회계 트랜잭션과 이벤트 전달 방식.
3. DB 마이그레이션 도구와 운영 기준.
4. JWT, refresh token, 로그아웃 정책.
5. 상품 이미지 저장소와 접근 정책.
6. 출고, 송장, 배송 상태의 경계와 책임.
