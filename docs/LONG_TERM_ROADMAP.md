# v0.3 이후 장기 로드맵

기준 버전: `v0.2.9`

v0.3.0 이후 장기 로드맵은 실무 ERP의 핵심 흐름을 축소 구현한 AI 커머스 ERP 포트폴리오 완성을 목표로 한다. 단순 쇼핑몰 화면이 아니라 상품, 주문, 결제, 재고, 배송, 회계, 운영 분석, AI 기능이 하나의 데이터 흐름으로 연결되는 관리자 중심 ERP 경험을 만든다.

## 전체 방향

```text
상품/전시 -> 장바구니/주문 -> 결제/환불 -> 재고/입고 -> 배송/반품
                                      -> 회계/정산 -> 분석/AI 운영
```

v0.1은 MVP 이후 mock 제거와 실제 API 연결을 완료했고, v0.2는 인증/결제/미디어/리뷰/DB/CI/알림/운영 분석 기반을 안정화했다. v0.3 이후는 실무 ERP 흐름을 포트폴리오 수준으로 확장한다.

## 상세 방향

### v0.3.0 상품/전시/운영관리

- 상품 상태, 전시 상태, 판매 상태를 분리한다.
- 카테고리, 옵션, 대표 이미지, 재고 상태를 관리자 운영 관점에서 정리한다.
- 상품 목록 필터, 대량 상태 변경, 운영 메모 등 실무형 관리 기능을 검토한다.

### v0.4.0 인사관리와 권한관리

- 직원, 부서, 직무, 권한 그룹의 최소 모델을 만든다.
- 현재 `USER`, `MANAGER`, `ADMIN`, `SUPER_ADMIN` 권한을 실무 역할 기반으로 세분화한다.
- 관리자 작업 이력을 리뷰 외 주요 운영 도메인으로 확장한다.

### v0.5.0 재고관리, 생산 입고, 바코드 자동화

- 창고 재고, 예약 재고, 가용 재고의 정합성을 강화한다.
- 입고 유형을 구매 입고, 생산 입고, 조정 입고 등으로 나눈다.
- 상품/입고/출고 단위 바코드 생성과 스캔 처리 기준을 만든다.

### v0.6.0 유통관리

- 택배사 연동 인터페이스를 설계한다.
- 송장번호 생성, 출력, 배송 상태 갱신 흐름을 구현한다.
- 출고 지시, 송장 출력, 배송 완료까지의 운영 흐름을 연결한다.

### v0.7.0 회계관리 고도화

- 매입, 매출, 환불, 입고 금액을 회계 이벤트로 정리한다.
- 정산, 마감, 리포트 기준을 고도화한다.
- 복식부기는 포트폴리오 범위에 맞춰 축소 구현하거나 문서 기준으로 분리한다.

### v0.8.0 AI 데이터셋과 모델 학습 구조

- 상품, 리뷰, 재고, 주문 데이터를 학습 가능한 dataset으로 정리한다.
- 전처리, 학습, 평가, `.pt` 모델 산출 경로를 구축한다.
- 운영 DB와 학습 데이터셋의 분리 기준을 문서화한다.

### v0.9.0 AI 기능과 최종 포트폴리오 릴리스

- AI 추천, 수요 예측, 리뷰 분석, 이상 주문 탐지 기능을 관리자 운영 화면에 연결한다.
- AI 결과는 운영자가 이해할 수 있는 이유/근거와 함께 표시한다.
- 최종 포트폴리오 문서, 데모 흐름, 릴리스 체크리스트를 정리한다.

## 우선순위 판단 기준

1. 금액, 재고, 개인정보 오류를 줄이는 안정성
2. 주문부터 사후 처리까지 끊기지 않는 운영 흐름
3. 실무 ERP 포트폴리오로 설명 가능한 도메인 완성도
4. 반복 업무를 줄이는 자동화
5. AI 기능에 필요한 데이터 품질과 추적 가능성

## 변경 규칙

- 버전 중간에 범위가 커지면 다음 버전 후보로 이관하고 이유를 체크리스트에 남긴다.
- API, DB, 권한, 회계, 재고 모델 변경은 관련 문서를 함께 갱신한다.
- 각 버전은 브랜치, 체크리스트, 검증 결과, 태그를 남긴다.
