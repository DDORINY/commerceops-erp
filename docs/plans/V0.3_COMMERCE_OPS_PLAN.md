# v0.3 상품/전시/운영관리 고도화 계획

기준 버전: `v0.3.0`
작업 브랜치: `v0.3.0-commerce-ops-plan`

v0.3은 기능 구현보다 먼저 상품, 전시, 운영관리의 데이터 모델과 화면 흐름을 실무 ERP 포트폴리오 기준으로 정리한다. 목표는 현재 상품 CRUD 중심 구조를 상품 마스터, 상품 상태, 전시 관리, 상세페이지 CMS, 배너 CMS, 운영 이력까지 확장할 수 있는 단계별 작업으로 나누는 것이다.

## v0.3 전체 방향

- 상품 마스터 정보를 실무 운영에 필요한 필드로 확장한다.
- 상품 상태, 전시 상태, 판매 상태를 분리한다.
- 상품 상세페이지를 블록 기반 에디터/CMS 구조로 계획한다.
- 카테고리와 상단 네비를 관리자 설정 기반으로 전환한다.
- 메인 배너를 관리자 CMS 데이터로 렌더링하도록 계획한다.
- 관리자 상품 운영 UX에 필터, 대량 변경, 운영 메모, 상태 이력을 추가한다.
- 사용자 상품 상세에서는 재고 상태와 구매 가능 여부를 더 명확하게 표시한다.

## v0.3.1 상품 마스터 고도화

진행 상태: `v0.3.1-product-catalog-master`에서 구현 완료.

목적: 현재 기본 상품 필드를 실무 커머스 운영에 필요한 상품 마스터 구조로 확장한다.

포함 범위:
- 상품코드, 브랜드, 제조사, 모델명, 원산지
- 정상가, 판매가, 할인가, 매입가, 마진율
- 검색 키워드, 상품 태그
- 판매 시작일/종료일
- 배송 정보
- SEO title, SEO description, SEO keywords

제외 범위:
- 외부 마켓 상품 매핑
- 다국어 상품명/번역
- 복잡한 가격 정책 엔진

예상 변경 파일/도메인:
- `backend/domain/product/entity/Product.java`
- `backend/domain/product/dto/*`
- `backend/domain/product/service/ProductService.java`
- `frontend/src/lib/services/productService.ts`
- `frontend/src/app/admin/products/**`

DB 변경:
- `products` 컬럼 확장
- `product_tags` 별도 테이블은 후속 AI 태그 추천/검색 고도화 시 검토
- `searchKeywords`, `tags`, `seoKeywords`는 v0.3.1에서 TEXT 구분자 문자열로 저장

API 변경 후보:
- `GET /api/products`
- `GET /api/products/{productId}`
- `GET /api/admin/products`
- `GET /api/admin/products/{productId}`
- `POST /api/admin/products`
- `PATCH /api/admin/products/{productId}`

프론트 화면 변경 후보:
- 관리자 상품 등록/수정 폼
- 관리자 상품 목록 컬럼/필터
- 사용자 상품 상세 SEO 표시 기준

완료 조건:
- 상품 마스터 필드가 저장/수정/조회된다.
- 관리자 상품 등록/수정 화면에서 신규 필드 입력이 가능하다.
- 사용자 화면은 기존 상품 조회가 깨지지 않는다.

검증 기준:
- frontend lint/build
- backend test
- 상품 등록/수정/조회 수동 확인

- 외부 WYSIWYG 플러그인 대규모 도입

프론트 화면 변경 후보:

- 사용자 상품 상세에서 노출 블록만 순서대로 보인다.

검증 기준:
- 블록 타입별 렌더링 확인

## v0.3.2 Implementation Note

- Status: v0.3.2 minimal implementation completed.
- Detail blocks are stored in `product_detail_blocks`.
- Admins can edit product detail blocks from the admin product edit page.
- Customer product detail pages render only `visible=true` blocks by `sortOrder`.
- HTML blocks are treated as admin-authored internal CMS content in v0.3.2. A sanitizer or allow-list policy should be added before broader operational use.

## v0.3.3 Implementation Note

- Status: v0.3.3 minimal implementation completed.
- `Category` now supports parent/depth/sortOrder/active/visibleInNav/slug.
- `GET /api/categories/navigation` powers the shop header navigation with active and nav-visible categories only.
- `/admin/categories` provides a basic admin screen for category creation, parent selection, sorting, active toggle, and navigation visibility.
- Product create/update category selection continues to use active categories from `GET /api/categories`; inactive categories are hidden from new UI selection but existing product references remain valid.

## v0.3.4 Implementation Note

- Status: v0.3.4 minimal implementation completed.
- `MainBanner` stores title/subtitle/description/imageUrl/linkUrl/position/sortOrder/active/startsAt/endsAt.
- Public `GET /api/banners` returns only active banners within the visible period. Null `startsAt` or `endsAt` is treated as an open-ended period.
- Admin `/api/admin/banners` supports list/detail/create/update/deactivate. Delete is implemented as `active=false`.
- Admin banner image handling is URL-based in v0.3.4. Dedicated banner image upload, resizing, CDN/S3, and click analytics remain future scope.

## v0.3.5 진행 메모

- v0.3.5에서 상품 판매 상태와 전시 상태를 분리했다.
