# 장기 로드맵

기준 버전: `v0.9.0`

CommerceOps ERP는 실무 ERP의 핵심 흐름을 축소 구현한 AI 커머스 ERP 포트폴리오 완성을 목표로 한다. 단순 쇼핑몰이 아니라 상품, 주문, 결제, 재고, 배송, 회계, 운영 분석, AI 기능이 하나의 데이터 흐름으로 연결되는 관리자 중심 ERP 경험을 만든다.

## 전체 데이터 흐름

```text
상품/전시 -> 장바구니/주문 -> 결제/환불 -> 재고/입고 -> 출고/배송/반품
                                      -> 회계/정산 -> 데이터셋/학습 -> AI 운영
```

## v0.3 상품/전시/운영관리

- 상품 마스터 필드를 실무 운영 기준으로 확장했다.
- 상품 상세페이지를 블록 기반 CMS 구조로 전환했다.
- 카테고리와 상단 네비를 관리자 설정 기반으로 렌더링하도록 정리했다.
- 메인 배너 CMS를 추가했다.
- 상품 판매 상태, 전시 상태, 구매 가능 여부를 분리했다.
- 관리자 상품 목록 필터, 대량 상태 변경, 운영 메모, 상태 변경 이력을 추가했다.
- 사용자 메인/상품 목록/상품 상세에 관리자 전시 데이터를 반영했다.

## v0.4 인사관리/권한관리

- 부서, 직급, 직원 프로필 모델을 추가했다.
- 직원 관리 API와 관리자 화면을 구현했다.
- 권한 그룹, 사용자 권한 그룹, 메뉴/기능 권한 매트릭스를 구현했다.
- 관리자 사이드바를 effective permission 기반으로 전환했다.
- 주요 관리자 API에 permission code 기반 세부 권한 검증을 적용했다.
- 관리자 작업 이력과 감사 로그를 확장했다.
- 사업자 설정과 약관/정책 버전 관리 기반을 추가했다.

## v0.5 재고관리/생산 입고/바코드

- SKU와 바코드 마스터를 도입했다.
- 생산 입고 흐름을 추가했다.
- 바코드 라벨 미리보기와 출력 이력을 구성했다.
- 바코드 기반 재고 조회, 입고, 출고 흐름을 추가했다.
- 창고별 재고 실사와 차이 조정 흐름을 구현했다.
- 창고 위치/로케이션과 위치별 SKU 재고 조회 기반을 추가했다.
- SKU 또는 SKU+창고 단위 안전재고 알림을 구현했다.

## v0.6 유통관리/송장/배송

- 출고 지시와 출고 항목 모델을 구현했다.
- 주문 기반 출고 지시 생성 흐름을 구현했다.
- 출고 항목과 SKU/바코드를 연결했다.
- 택배사와 배송 방법을 관리자에서 관리할 수 있게 했다.
- 송장번호 생성 또는 수동 입력 흐름을 정리했다.
- 송장 라벨 미리보기와 출력 이력을 기록했다.
- 배송 상태와 배송 추적 이벤트를 저장하도록 구현했다.
- 반품 배송 정보와 반품 배송비 정보를 연결했다.
- 바코드 기반 출고 검수와 출고 완료 기준을 정리했다.

- 환불, 반품 배송비, 택배비 매입 회계 반영 흐름을 추가했다.
- 정산 배치와 기간 마감 구조를 구현했다.
- 회계 리포트 UI 후보와 정합성 점검 API를 추가했다.
- 회계 권한과 감사 로그 기록 기준을 정리했다.

## v0.8 AI 데이터셋과 모델 학습 구조

- 상품, 리뷰, 주문, 재고, 배송, 회계/정산 데이터를 AI 학습 후보 데이터셋으로 정리했다.
- AI 데이터셋 카탈로그/export API를 구현했다.
- 개인정보 마스킹 유틸을 추가했다.
- 상품/리뷰, 주문/수요, 재고/배송, 회계/정산 데이터셋을 추가했다.
- `ai/` 학습 파이프라인 구조를 만들었다.
- 포트폴리오용 합성 샘플 데이터와 demo config를 추가했다.
- 수요 예측과 리뷰 감성 baseline `.pt` 모델 생성 흐름을 검증했다.
- 평가 리포트 JSON/Markdown 생성 흐름을 검증했다.
- 모델 파일과 대용량 데이터셋 산출물은 Git 추적 대상에서 제외했다.

## v0.9 AI 운영 기능과 최종 릴리스

- v0.9.0에서 AI 운영 기능 계획을 확정한다.
- AI 추천, 수요 예측, 리뷰 분석, 이상 주문 탐지, 재고/정산 리스크 알림을 관리자 운영 화면에 연결한다.
- AI 결과는 점수만 보여주지 않고 근거 피처와 설명 문구를 함께 제공한다.
- 실제 운영 데이터가 없는 포트폴리오 환경이므로 합성 데이터와 baseline 모델로 재현 가능한 데모를 구성한다.
- 최종 릴리스에서는 데모 시나리오, 실행 명령, 구현 범위, 제외 범위를 명확히 정리한다.
- v0.9 세부 계획은 [V0.9_AI_OPERATIONS_PLAN.md](V0.9_AI_OPERATIONS_PLAN.md)를 기준으로 한다.

## 우선순위 판단 기준

1. 금액, 재고, 개인정보 오류를 줄이는 안정성.
2. 주문 이후 출고, 배송, 반품, 정산까지 끊기지 않는 운영 흐름.
3. 포트폴리오에서 실무 ERP 경험으로 설명 가능한 화면과 데이터 구조.
4. 반복 업무를 줄이는 자동화.
5. AI 기능에 필요한 데이터 품질과 추적 가능성.

## 운영 규칙

- 각 작업은 버전 브랜치, 체크리스트, 검증 결과, 태그를 남긴다.
- 버전 중간에 범위가 커지면 다음 버전 후보로 이관하고 이유를 체크리스트에 기록한다.
- API, DB, 권한, 회계, 재고, AI 구조 변경은 관련 문서를 함께 갱신한다.
- 모든 Markdown 문서는 한국어를 기본으로 작성한다.
