# 장기 로드맵

기준 버전: `v0.7.0`

CommerceOps ERP는 실무 ERP의 핵심 흐름을 축소 구현한 AI 커머스 ERP 포트폴리오 완성을 목표로 한다. 단순 쇼핑몰 화면이 아니라 상품, 주문, 결제, 재고, 배송, 회계, 운영 분석, AI 기능이 하나의 데이터 흐름으로 연결되는 관리자 중심 ERP 경험을 만든다.

## 전체 방향

```text
상품/전시 -> 장바구니/주문 -> 결제/환불 -> 재고/입고 -> 출고/배송/반품
                                      -> 회계/정산 -> 분석/AI 운영
```

## v0.3 상품/전시/운영관리

- 상품 마스터 필드를 실무 운영 기준으로 확장했다.
- 상품 상세페이지를 블록 기반 CMS 구조로 전환했다.
- 카테고리와 상단 네비를 관리자 설정 기반으로 렌더링하도록 정리했다.
- 메인 배너 CMS를 추가했다.
- 상품 판매 상태, 전시 상태, 구매 가능 여부를 분리했다.
- 관리자 상품 목록 필터, 대량 상태 변경, 운영 메모, 상태 변경 이력을 추가했다.
- 사용자 메인/상품 목록/상품 상세에 관리자 전시 데이터를 반영했다.

## v0.4 인사관리/권한관리

- 부서, 직급, 직원 프로필 모델을 추가했다.
- 직원 관리 API와 관리자 화면을 구현했다.
- 권한 그룹과 사용자 권한 그룹 매핑을 추가했다.
- 메뉴별/기능별 권한 매트릭스를 구현했다.
- 관리자 사이드바를 effective permission 기반으로 전환했다.
- 주요 관리자 API에 permission code 기반 세부 권한 검증을 적용했다.
- 관리자 작업 이력과 감사 로그를 확장했다.
- 사업자 설정과 약관/정책 버전 저장 구조를 구현했다.

## v0.5 재고관리/생산 입고/바코드 자동화

- SKU와 바코드 마스터를 도입했다.
- 생산 주문과 생산 완료 입고 흐름을 추가했다.
- 바코드 라벨 미리보기와 출력 이력 기반을 만들었다.
- 바코드 기반 재고 조회, 입고, 출고 흐름을 추가했다.
- 창고별 재고 실사와 차이 조정 흐름을 구현했다.
- 창고 위치/로케이션과 위치별 SKU 재고 조회 기반을 추가했다.
- SKU 또는 SKU+창고 단위 안전재고 알림을 구현했다.
- v0.5.9에서 v0.5 전체 범위를 검증하고 v0.6 유통관리로 이관했다.

## v0.6 유통관리/송장/배송

## v0.7 회계관리 고도화

v0.7.0에서 회계관리 고도화 계획을 확정했으며, 실제 구현은 v0.7.1부터 진행한다.

- 매입, 매출, 환불, 배송비를 회계 이벤트 후보로 정리했다.
- 배송비와 환불 배송비를 주문/배송/반품 흐름과 연결하는 기준을 확정했다.
- 정산 배치와 기간 마감의 최소 구조를 계획했다.
- 월별/택배사별/상품별 회계 리포트 후보를 정리했다.
- 회계 권한과 감사 로그 기록 기준을 정리했다.
- 복식부기, 세무 신고, 외부 회계 프로그램 연동은 제외 범위로 분리했다.

## v0.8 AI 데이터셋과 모델 학습 구조

- 상품, 리뷰, 재고, 주문 데이터를 학습 가능한 dataset으로 정리한다.
- AI 기능에 필요한 전처리, 학습, 평가, 모델 저장 경로를 구축한다.
- 운영 DB와 학습 데이터셋을 분리하는 기준을 문서화한다.
- `.pt` 모델 산출 구조를 정리한다.
- AI 코드, 대용량 dataset/model Git 제외 기준, FastAPI 추론 서버, AWS 1대 서버 배포 전략은 [AI_DEVELOPMENT_DEPLOYMENT_PLAN.md](../ai/AI_DEVELOPMENT_DEPLOYMENT_PLAN.md)를 따른다.

## v0.9 AI 운영 기능과 최종 릴리스

- AI 추천, 수요 예측, 리뷰 분석, 이상 주문 탐지를 관리자 운영 화면에 연결한다.
- AI 결과에 대한 근거와 운영자가 이해할 수 있는 설명을 함께 제공한다.
- 최종 포트폴리오 문서, 데모 흐름, 릴리스 체크리스트를 정리한다.
- LLM은 핵심 모델이 아니라 제한적인 관리자 보조 기능으로 두고, 사용량 제한/캐싱/로그 정책을 적용한다.
- v0.9 세부 계획은 [V0.9_AI_OPERATIONS_PLAN.md](V0.9_AI_OPERATIONS_PLAN.md)에 정리한다.

## 우선순위 판단 기준

1. 금액, 재고, 개인정보 오류를 줄이는 안정성.
2. 주문 이후 출고, 배송, 환불, 정산까지 끊기지 않는 운영 흐름.
3. 포트폴리오에서 실무 ERP 경험으로 설명 가능한 화면과 데이터 구조.
4. 반복 업무를 줄이는 자동화.
5. AI 기능에 필요한 데이터 품질과 추적 가능성.

## 운영 규칙

- 각 작업은 버전 브랜치, 체크리스트, 검증 결과, 태그를 남긴다.
- 버전 중간에 범위가 커지면 다음 버전 후보로 이관하고 이유를 체크리스트에 기록한다.
- API, DB, 권한, 회계, 재고 모델 변경은 관련 문서를 함께 갱신한다.
- 모든 MD 문서는 한국어를 기본으로 작성한다.
## v0.8 AI 데이터셋 및 학습 기반

v0.8은 v0.7까지 축적한 상품, 주문, 재고, 배송, 리뷰, 문의, 회계/정산 데이터를 AI 학습 데이터셋으로 전환하는 단계다. 개인정보 마스킹, 데이터 스냅샷, 학습 피처 정의, `.pt` 모델 학습 파이프라인을 우선 정리하고, 실제 운영 화면 자동화는 v0.9로 넘긴다.

v0.8.0에서는 전체 계획을 확정하고, v0.8.1부터 데이터 추출, 마스킹, 도메인별 데이터셋, 학습 파이프라인, 평가 리포트를 순차 구현한다.
