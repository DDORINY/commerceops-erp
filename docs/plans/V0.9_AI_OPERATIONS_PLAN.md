# v0.9 AI 운영 기능 및 최종 포트폴리오 릴리스 계획

## 목표

v0.9는 v0.8에서 만든 데이터셋, 마스킹, 학습, 평가 기반을 관리자 운영 화면에 연결하는 단계다. AI 기능은 운영자가 이해하고 검토할 수 있는 보조 기능으로 제공하며, 자동 의사결정이나 외부 과금형 AI 의존은 최소화한다.

## v0.9 전체 방향

- AI 결과는 관리자 화면에서 근거와 함께 표시한다.
- 운영자가 확인하고 실행하는 구조를 우선한다.
- 예측/추천 결과는 업무 자동 처리보다 의사결정 보조로 사용한다.
- 모델 파일과 데이터셋은 Git이 아닌 외부 스토리지 또는 로컬 아티팩트로 관리한다.
- 실패 시 기존 ERP 기능이 깨지지 않도록 AI 기능은 선택적 보조 기능으로 분리한다.

## 주요 기능 후보

### AI 추천

- 상품별 추천 후보
- 고객/주문 맥락 기반 추천 후보
- 추천 근거: 카테고리, 가격대, 리뷰 평점, 태그, 구매 이력 후보
- 운영 자동 적용은 제외하고 관리자 확인 후 활용

### 수요 예측

- 상품별 예상 주문 수
- 기간별 예상 매출
- 재고 부족 위험 후보
- 입력 피처: 주문 시간, 요일, 금액, 할인, 재고, 판매 상태

### 리뷰 분석

- 리뷰 감성 후보
- 리뷰 키워드 후보
- 숨김 검토 후보
- 개인정보 마스킹 유지

### 이상 주문 탐지

- 비정상 고액 주문 후보
- 반복 환불/취소 후보
- 배송 지연과 환불 연계 후보
- 회계 정합성 이슈와 연결된 주문 후보

### 재고/정산 리스크

- 안전재고 하회 가능성
- 입고 지연 가능성
- 정산 누락 가능성
- 배송비/환불 비용 이상 후보

## 제외 범위

- 완전 자동 발주
- 완전 자동 가격 변경
- 외부 LLM 중심 운영 자동화
- 대규모 MLOps 플랫폼
- 실시간 고가용 추론 서버
- 외부 AI API 과금 연동
- 개인정보 원문 기반 학습
- 법적 판단/회계 확정 자동화

## v0.8에서 이관된 기반

- AI 데이터셋 카탈로그/export API
- 개인정보 마스킹 유틸
- 상품/리뷰, 주문/수요, 재고/배송, 회계/정산 데이터셋
- baseline 학습 스크립트
- 모델 평가/리포트 스크립트
- 모델/데이터셋 산출물 Git 제외 기준

## 검증 기준

- AI 기능 실패가 기존 ERP 화면을 깨뜨리지 않아야 한다.
- 관리자 권한과 permission 정책을 유지해야 한다.
- 개인정보 원문이 AI 리포트와 로그에 남지 않아야 한다.
- 추천/예측 결과에는 입력 근거를 함께 제공해야 한다.
- 최종 릴리스 문서는 데모 시나리오와 구현 범위를 명확히 구분해야 한다.
