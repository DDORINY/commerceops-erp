# v0.9 AI 운영 기능 및 최종 포트폴리오 릴리스 계획

## 목표

v0.9는 v0.8에서 만든 AI 데이터셋, 개인정보 마스킹, baseline 학습, 평가 리포트 기반을 관리자 운영 화면에 연결하는 단계다.

이 프로젝트는 실제 서비스 운영이 아니라 포트폴리오 목적이므로, 실제 고객 데이터를 수집하거나 운영 의사결정을 자동화하지 않는다. 대신 개인정보가 없는 합성 데이터와 baseline 모델을 기반으로 “AI 커머스 ERP가 어떤 데이터 흐름과 운영 화면을 가질 수 있는지”를 재현 가능하게 보여준다.

## 핵심 방향

- AI 기능은 기존 ERP 기능을 대체하지 않고 관리자 의사결정 보조 기능으로 제공한다.
- AI 결과에는 점수뿐 아니라 근거 피처와 해석 문구를 함께 제공한다.
- 추천/예측/분석 결과는 자동 적용하지 않고, 관리자가 검토 후 판단하는 흐름을 우선한다.
- AI API나 화면이 실패해도 주문, 상품, 재고, 회계 같은 기존 ERP 화면은 깨지지 않아야 한다.
- 실제 운영 데이터 대신 합성 샘플 데이터와 관리자 export 구조를 기준으로 포트폴리오 데모를 구성한다.
- `.pt`, 평가 리포트, export 원본 파일은 Git에 커밋하지 않고 재현 가능한 스크립트/config/샘플 데이터만 관리한다.

## v0.8에서 이관된 기반

- AI 데이터셋 카탈로그/export API
- `AI_DATASET_EXPORT` 권한
- 개인정보 마스킹 유틸
- 상품/리뷰, 주문/수요, 재고/배송, 회계/정산 데이터셋
- `train_baseline.py` 학습 스크립트
- `evaluate_baseline.py` 평가 스크립트
- 포트폴리오용 합성 샘플 데이터
- 수요 예측과 리뷰 감성 baseline `.pt` 생성 흐름
- 모델/데이터셋/리포트 산출물 Git 제외 기준

## 공통 설계 기준

### AI 결과 공통 응답 후보

```json
{
  "id": "string",
  "targetType": "PRODUCT",
  "targetId": 1,
  "title": "string",
  "score": 0.82,
  "riskLevel": "LOW|MEDIUM|HIGH",
  "reason": "관리자가 이해할 수 있는 설명",
  "features": {
    "priceLevel": 5.2,
    "reviewRating": 4.7
  },
  "modelName": "commerceops_demo_demand_baseline",
  "generatedAt": "2026-07-13T00:00:00"
}
```

### riskLevel 기준

- `LOW`: 참고 수준
- `MEDIUM`: 운영자가 확인할 필요가 있는 수준
- `HIGH`: 우선 확인 대상

### 권한 기준

- AI 운영 화면 조회: `MANAGER`, `ADMIN`, `SUPER_ADMIN`
- AI 설정/재생성/모델 파일 기준 변경: `ADMIN`, `SUPER_ADMIN`
- 세부 permission 후보:
  - `AI_DATASET_EXPORT`
  - `AI_RECOMMENDATION_READ`
  - `AI_FORECAST_READ`
  - `AI_REVIEW_ANALYSIS_READ`
  - `AI_ANOMALY_READ`
  - `AI_RISK_ALERT_READ`
  - `AI_REPORT_READ`
  - `AI_OPERATIONS_MANAGE`

v0.9에서는 기존 role 기반 접근과 permission code 기반 메뉴 노출을 병행한다. 완전한 모델 운영 권한 체계는 최종 포트폴리오 범위 내에서 최소화한다.

## 포트폴리오 설명 기준

> CommerceOps ERP는 실제 운영 데이터를 사용하지 않고, 개인정보가 없는 합성 데이터와 관리자 export 구조를 통해 AI 학습/평가 파이프라인을 재현합니다. v0.9에서는 이 기반을 관리자 운영 화면에 연결해 추천, 수요 예측, 리뷰 분석, 이상 주문 탐지, 재고/정산 리스크 알림을 포트폴리오 데모 수준으로 구현합니다.

## v0.9.1 진행 상태

- AI 운영 공통 insight 응답 구조를 `AiInsightResponse`로 추가했다.
- 관리자 AI 운영 overview/health API를 `/api/admin/ai/overview`, `/api/admin/ai/health`로 추가했다.
- 관리자 사이드바에 `AI 운영` 그룹과 `/admin/ai` 화면을 추가했다.
- v0.9 AI permission code와 메뉴 seed를 추가했다.
- v0.9.2 이후 세부 화면은 이 공통 응답 구조와 권한 기준을 확장해서 구현한다.

## v0.9.2 진행 상태

- 상품 추천 후보 API를 `/api/admin/ai/recommendations/products`로 추가했다.
- 추천 후보는 상품 전시/판매 상태, 재고, 태그, 검색 키워드, 이미지, 판매가를 기준으로 rule-based 점수를 계산한다.
- 관리자 화면 `/admin/ai/recommendations`에서 추천 후보와 근거를 확인할 수 있다.
- 추천 결과는 자동 전시나 개인화 추천에 반영하지 않고 관리자 검토용 참고 지표로만 사용한다.

## v0.9.3 진행 상태

- 수요 예측 API를 `/api/admin/ai/forecasts/demand`로 추가했다.
- 예측 후보는 현재 재고, 안전재고, 태그, 검색 키워드를 기준으로 rule-based 재고 소진 위험을 계산한다.
- 관리자 화면 `/admin/ai/demand-forecast`에서 예측 상태와 근거를 확인할 수 있다.
- 예측 결과는 자동 발주/생산 지시에 반영하지 않고 관리자 보충 판단용 참고 지표로만 사용한다.

## v0.9.4 진행 상태

- 리뷰 분석 API를 `/api/admin/ai/reviews/analysis`로 추가했다.
- 분석 후보는 리뷰 평점과 마스킹된 본문을 기준으로 rule-based 감성 후보를 계산한다.
- 관리자 화면 `/admin/ai/review-analysis`에서 감성 후보와 분석 근거를 확인할 수 있다.
- 분석 결과는 자동 숨김/제재에 반영하지 않고 관리자 검토용 참고 지표로만 사용한다.

## v0.9.5 진행 상태

- 이상 주문 탐지 API를 `/api/admin/ai/anomalies/orders`로 추가했다.
- 탐지 후보는 주문 금액, 할인율, 주문/결제 상태 불일치 기준으로 rule-based 위험 점수를 계산한다.
- 관리자 화면 `/admin/ai/order-anomalies`에서 이상 주문 후보와 탐지 근거를 확인할 수 있다.
- 탐지 결과는 자동 주문 차단에 반영하지 않고 관리자 검토용 참고 지표로만 사용한다.

## v0.9.6 진행 상태

- 재고 리스크 API를 `/api/admin/ai/risks/inventory`로 추가했다.
- 정산 리스크 API를 `/api/admin/ai/risks/settlement`로 추가했다.
- 관리자 화면 `/admin/ai/risk-alerts`에서 재고/정산 리스크 후보와 근거를 확인할 수 있다.
- 리스크 결과는 자동 입고나 정산 마감에 반영하지 않고 관리자 확인용 참고 지표로만 사용한다.
