# v1.0 최종 포트폴리오 릴리스 계획

## 목표

v1.0은 새로운 대형 기능을 추가하는 단계가 아니라, v0.1 ~ v0.9에서 구현한 CommerceOps ERP를 포트폴리오 제출과 시연이 가능한 최종 릴리스 상태로 정리하는 단계다.

최종 목표는 “실무 ERP의 핵심 흐름을 축소 구현한 AI 커머스 ERP”를 코드, 문서, 데모 데이터, 실행 방법, 검증 결과까지 일관되게 설명할 수 있게 만드는 것이다.

## 현재 기준

- 기준 완료 버전: `v0.9.9`
- 구현 축:
  - 인증/세션/권한
  - 상품/전시/배너/카테고리
  - 주문/결제/환불
  - 고객/문의/리뷰/알림
  - 직원/권한/감사 로그
  - 재고/SKU/바코드/생산 입고
  - 출고/송장/배송/반품
  - 회계/정산/마감/리포트
  - AI 데이터셋/학습/평가
  - AI 운영 화면/데모 시나리오

## v1.0 작업 원칙

1. 신규 대형 기능 구현보다 제출 품질과 재현성을 우선한다.
2. README, 실행 가이드, 데모 시나리오, 문서 인덱스가 서로 충돌하지 않게 정리한다.
3. 실제 서비스 운영처럼 과장하지 않고 포트폴리오 범위와 제외 범위를 명확히 쓴다.
4. 실행 가능한 검증 명령과 수동 확인 항목을 문서에 남긴다.
5. `.env`, 로컬 DB 정보, `.pt` 모델, 평가 리포트, 빌드 산출물은 Git에 포함하지 않는다.

## v1.0.1 최종 문서 정리

외부 평가자가 README에서 시작해 프로젝트 목적, 실행 방법, 핵심 기능, 데모 흐름, 제외 범위를 빠르게 이해할 수 있게 한다.

포함 범위:
- 루트 `README.md` 최신화
- `docs/README.md`, `docs/INDEX.md` 정합성 확인
- 포트폴리오 요약 문서 최신화
- `docs/operations/CURRENT_STATE.md` 기준 버전 갱신
- API/DB/프론트/백엔드 구조 문서의 오래된 버전 표현 정리
- AI 데모 문서 연결 확인

제외 범위:
- 기능 코드 수정
- DB migration 추가
- 새로운 API 추가

완료 조건:
- 주요 문서가 `v0.9.9` 완료 상태를 기준으로 설명한다.
- 포트폴리오 제출용 문서 시작점이 명확하다.
- 문서 공백 검증이 통과한다.

## v1.0.2 데모 데이터/계정 정리

면접이나 발표에서 동일한 화면을 재현할 수 있도록 데모 계정, 샘플 데이터, AI 데모 실행 기준을 정리한다.

포함 범위:
- 관리자/운영자/일반 사용자 데모 계정 기준 문서화
- 시연에 필요한 상품, 주문, 재고, 배송, 회계, AI 샘플 데이터 범위 정리
- AI 합성 데이터와 `.pt` 모델 재생성 명령 정리
- 민감 정보와 로컬 환경 파일 제외 기준 확인

제외 범위:
- 실제 개인정보 포함 seed
- 운영 DB 덤프 커밋
- 대량 이미지/모델 산출물 커밋

완료 조건:
- 데모 환경을 재현하기 위한 계정/데이터/명령이 문서화된다.
- Git에 포함하지 않을 산출물 기준이 명확하다.

## 최종 포트폴리오 설명 문구

> CommerceOps ERP는 쇼핑몰과 ERP 운영 흐름을 하나의 데이터 흐름으로 연결한 AI 커머스 ERP 포트폴리오입니다. 상품 전시에서 주문, 결제, 재고, 바코드, 출고, 배송, 회계, 권한, 감사 로그, AI 운영 화면까지 실무 ERP의 핵심 흐름을 축소 구현했습니다. 실제 운영 데이터 대신 합성 샘플 데이터와 baseline 모델을 사용해 AI 데이터셋, 학습, 평가, 운영 화면 연결 구조를 재현 가능하게 구성했습니다.
