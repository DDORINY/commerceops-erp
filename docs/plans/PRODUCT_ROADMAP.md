# CommerceOps ERP 제품 로드맵

기준 버전: `v1.0.0`

CommerceOps ERP는 단순 쇼핑몰 화면이 아니라 상품, 전시, 주문, 결제, 재고, 생산 입고, 바코드, 출고, 배송, 반품, 권한, 감사 로그, 회계, AI 운영 기능을 하나의 데이터 흐름으로 연결하는 관리자 중심 ERP 포트폴리오를 목표로 한다.

## 제품 목표

```text
상품/전시 -> 장바구니/주문 -> 결제/환불 -> 재고/생산 입고/바코드
                                      -> 출고/송장/배송/반품
                                      -> 회계/정산
                                      -> AI 데이터셋/AI 운영 기능
```

최종 목표는 실무 ERP의 핵심 흐름을 축소 구현해 포트폴리오로 설명 가능한 AI 커머스 ERP를 완성하는 것이다. 현재 `v0.9.9`까지 AI 운영 기능과 최종 데모 정리를 완료했으며, `v1.0`에서는 제출 가능한 최종 릴리스 품질을 정리한다.

## 로드맵 원칙

1. 기능 개수보다 실제 운영자가 끝까지 따라갈 수 있는 업무 흐름을 우선한다.
2. 금액, 재고, 개인정보, 권한과 관련된 변경은 감사 로그와 문서 기준을 함께 남긴다.
3. 화면, API, DB, 권한, 테스트, 문서가 함께 정리되어야 완료로 본다.
4. 범위가 커진 기능은 다음 버전으로 이관하고 이유를 체크리스트에 기록한다.
5. 아직 구현되지 않은 v0.7 이후 기능은 “예정”으로 구분하고, 완료된 것처럼 표현하지 않는다.

## 완료된 제품 축

### v0.3 상품/전시/운영관리

- 상품 마스터 필드를 실무 운영 기준으로 확장했다.
- 상품 상세페이지를 블록 기반 CMS 구조로 전환했다.
- 카테고리와 쇼핑몰 상단 네비를 관리자 설정 기준으로 렌더링한다.
- 메인 배너 CMS를 추가했다.
- 상품 판매 상태, 전시 상태, 구매 가능 여부를 분리했다.
- 상품 운영 메모와 상태 변경 이력을 기록한다.

### v0.4 인사관리/권한관리

- 직원, 부서, 직급, 직원 프로필 모델을 추가했다.
- 권한 그룹과 사용자 권한 그룹 매핑을 구현했다.
- 메뉴별/기능별 권한 매트릭스를 구현했다.
- 관리자 사이드바와 주요 관리자 API를 permission code 기준으로 세분화했다.
- 감사 로그를 요청 컨텍스트와 before/after JSON 기준으로 확장했다.
- 사업자 설정과 약관/정책 버전 저장 구조를 구현했다.

### v0.5 재고관리/생산 입고/바코드 자동화

- SKU/바코드 마스터를 도입했다.
- 생산 주문과 생산 완료 입고 흐름을 구현했다.
- 바코드 라벨 미리보기와 출력 이력을 기록한다.
- 바코드 기반 재고 조회, 입고, 출고를 제공한다.
- 재고 실사, 창고 위치, 안전재고 알림을 구현했다.

### v0.6 유통관리/송장/배송

- 주문 기반 출고 지시와 출고 품목을 구현했다.
- 택배사와 배송 방법을 관리자에서 관리한다.
- 송장번호 수동 입력, 내부 테스트용 자동 생성, 송장 라벨 미리보기, 출력 이력을 제공한다.
- 배송 상태와 추적 이벤트를 기록한다.
- 반품 배송 정보와 출고 바코드 검수 흐름을 추가했다.

## 도메인 상태 흐름

### 주문/배송

```text
PENDING -> PAID -> PREPARING -> SHIPPING -> COMPLETED
                      └───────────────> CANCELLED
```

배송은 출고 지시, 송장번호, 배송 상태 추적과 연결된다. 실제 택배사 API 자동 연동은 아직 예정 범위다.

### 반품/환불

```text
REQUESTED -> APPROVED -> 회수/검수 -> 재고 복구 -> 회계 반영 -> REFUNDED
          └-> REJECTED
```

현재는 반품 승인/거절, 반품 배송 정보, 재고/회계 연동 기반을 제공한다. 복잡한 교환/부분 반품 자동화는 후속 범위다.

### 회계/정산 예정 흐름

```text
결제/환불/배송비/입고 이벤트 -> 회계 거래 기록 -> 정산 배치 -> 마감 -> 리포트
```

`v0.7.x`에서는 기존 단일 회계 내역을 확장해 정산과 마감 기준을 정리한다.

## 로드맵 변경 규칙

- 버전 중간에 기능이 커지면 같은 버전에서 억지로 처리하지 않고 다음 버전 후보로 이관한다.
- API, DB, 권한, 회계, 재고 모델 변경은 관련 문서를 함께 갱신한다.
- 구현되지 않은 기능은 “예정” 또는 “후속 범위”로 표기한다.
- 모든 MD 문서는 한국어와 UTF-8 기준으로 관리한다.
