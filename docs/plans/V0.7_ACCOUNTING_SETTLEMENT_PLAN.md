# v0.7 회계관리 고도화 계획

기준 버전: `v0.6.9-release-verification-v07-handover`

문서 상태: v0.6 유통관리/송장/배송 구현 결과를 바탕으로 v0.7에서 다룰 회계관리 고도화 범위와 제외 범위를 정리한 계획 문서다. v0.7 기능은 아직 구현되지 않았으며, 이 문서는 v0.7.0 이후 작업 기준으로 사용한다.

## 목표

v0.7은 현재 단일 회계 내역 중심 구조를 확장해 매입, 매출, 환불, 배송비, 택배비, 정산, 마감, 회계 리포트를 실무 ERP 포트폴리오 수준으로 축소 구현하는 것을 목표로 한다.

## v0.6에서 넘어온 데이터 후보

- 주문 매출: 주문 생성, 결제 완료, 주문 상태 변경 기준 매출 인식 후보.
- 환불: 결제 취소/환불, 반품 승인, 주문 `REFUNDED` 상태 기준 환불 인식 후보.
- 배송비 매출: 주문 배송비 또는 배송 정책 기반 고객 청구 배송비 후보.
- 반품 배송비: `return_shipment_infos.shipping_fee`, `fee_payer` 기준 고객/회사 부담 구분 후보.
- 택배비 매입: `shipping_methods.default_fee`, 택배사/배송 방법 기준 내부 비용 후보.
- 출고 기준 매출 확정: 출고 검수 완료 또는 배송중 전환 시점 기준 매출 확정 후보.
- 송장/라벨/추적 비용: 실제 택배사 API/라벨 출력 비용은 후속 외부 연동 이후 연결 후보.

## DB 설계 후보

| 테이블 후보 | 목적 | 핵심 필드 후보 |
| --- | --- | --- |
| `accounting_ledgers` | 회계 원장 header | `id`, `ledger_number`, `period`, `status`, `closed_at`, `closed_by`, timestamps |
| `accounting_transactions` | 회계 거래 내역 | `id`, `type`, `amount`, `direction`, `reference_type`, `reference_id`, `occurred_at`, timestamps |
| `settlement_batches` | 정산 배치 | `id`, `period_start`, `period_end`, `status`, `total_sales`, `total_refunds`, `total_shipping_fee`, `closed_at` |
| `shipping_cost_entries` | 택배비/배송비 비용 후보 | `id`, `shipment_id`, `carrier`, `shipping_method_id`, `cost_amount`, `charge_amount`, `occurred_at` |
| `return_fee_entries` | 반품 배송비 회계 후보 | `id`, `return_id`, `fee_payer`, `amount`, `settlement_status`, timestamps |

## API 후보

- `GET /api/admin/accounting/ledgers`
- `POST /api/admin/accounting/ledgers/close`
- `GET /api/admin/accounting/transactions`
- `GET /api/admin/accounting/settlements`
- `POST /api/admin/accounting/settlements`
- `GET /api/admin/accounting/shipping-costs`
- `GET /api/admin/accounting/return-fees`
- `GET /api/admin/accounting/reports/summary`

## 프론트 화면 후보

- `/admin/accounting`: 원장, 거래 내역, 정산 내역 중심으로 고도화.
- `/admin/sales`: 주문/출고/배송 기준 매출 분석 보강.
- `/admin/settings`: 회계 마감 정책과 배송비 정산 정책 후보.
- `/admin/returns`: 반품 배송비 회계 반영 상태 표시 후보.
- `/admin/shipments`: 배송비/택배비 정산 후보 표시.

## 권한 정책 후보

- 조회: `ACCOUNTING_READ`
- 마감/정산 실행: `ACCOUNTING_CLOSE`
- 환불 실행: `PAYMENT_REFUND`
- 설정 변경: `SETTINGS_MANAGE`
- 감사 로그 조회: `AUDIT_LOG_READ`

## 제외 범위

- 완전한 복식부기 회계 시스템.
- 실제 세무 신고와 법정 서식.
- PG/택배사 실제 정산 파일 자동 수집.
- 외부 ERP/회계 프로그램 연동.
- 멀티 사업자/멀티 테넌트 정산.
