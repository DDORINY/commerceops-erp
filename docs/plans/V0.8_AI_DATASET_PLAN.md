# v0.8 AI 데이터셋 및 학습 기반 계획

## 목표

v0.8은 CommerceOps ERP에 축적된 상품, 주문, 결제, 환불, 반품, 재고, 배송, 리뷰, 문의, 회계/정산 데이터를 AI 학습용 데이터셋으로 정리하고, `.pt` 모델 학습 구조를 구축하는 단계다.

v0.8에서는 실제 운영 화면 자동화보다 데이터셋 품질, 개인정보 비식별화, 학습/검증 파이프라인, 모델 산출물 관리 기준을 먼저 확정한다. 추천, 수요 예측, 리뷰 분석, 이상 주문 탐지 같은 운영 기능 연결은 v0.9에서 진행한다.

## v0.8 전체 방향

- 운영 DB 원본을 직접 학습하지 않고 export 또는 snapshot 데이터를 사용한다.
- 개인정보와 민감 정보는 학습 데이터셋 생성 전에 제거하거나 마스킹한다.
- 데이터셋, 학습 코드, 모델 산출물, 평가 리포트의 버전 관계를 문서화한다.
- `.pt` 모델 파일은 Git에 커밋하지 않고 저장 위치와 관리 규칙만 문서화한다.
- v0.8은 AI를 붙이기 위한 기반이며, 관리자 화면의 실제 AI 자동화는 v0.9로 이관한다.

## 개인정보/보안 기준

- 이름, 이메일, 전화번호, 주소, 토큰, 비밀번호, 상세 배송지는 학습 데이터셋에 포함하지 않는다.
- 사용자 식별자는 직접 ID 대신 해시 또는 내부 surrogate key 사용을 검토한다.
- 리뷰/문의 본문은 개인정보 포함 가능성이 있으므로 마스킹 또는 샘플링 기준을 둔다.
- export 파일은 Git 추적 대상에서 제외한다.
- 모델 파일(`*.pt`, `*.pth`, `*.onnx`)은 Git 추적 대상에서 제외한다.
- 학습 로그에는 민감 원문을 출력하지 않는다.

## 학습 파이프라인 후보 구조

```text
    processed/      # Git 제외, 전처리 결과
  models/
    checkpoints/    # Git 제외, .pt 산출물
  scripts/
    export_*.py
    preprocess_*.py
    train_*.py
    evaluate_*.py
  configs/
    dataset.yaml
    train.yaml
  reports/
    evaluation_*.md
```
