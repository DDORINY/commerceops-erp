# v0.4 인사관리/권한관리 계획

기준 버전: `v0.4.0-hr-permission-plan`

문서 상태: v0.3.9에서 이관한 인사관리/권한관리 후보를 v0.4.0에서 확정한 계획 문서다. v0.4.1부터 기능 구현을 진행하며, 이 문서는 v0.4 전체 범위와 제외 범위, DB/API/프론트/권한 설계 기준을 기록한다.

## 목표

v0.4의 목표는 최고관리자가 직원, 부서, 직급, 역할, 권한 그룹을 관리하고, 관리자 화면과 API 접근을 세분화된 권한 기준으로 통제할 수 있는 기반을 만드는 것이다.

기존 `USER`, `MANAGER`, `ADMIN`, `SUPER_ADMIN` role은 유지하되, 실무 운영에 필요한 세부 권한은 permission code와 permission group 기준으로 확장한다.

## 권한 정책

- 기존 role은 유지한다.
- `SUPER_ADMIN`은 모든 권한을 보유한 것으로 간주한다.
- `ADMIN`은 운영 권한 중심으로 사용한다.
- `MANAGER`는 조회 중심 권한으로 시작한다.
- 세부 기능 실행은 permission code 기준으로 확장한다.
- 관리자 사이드바 노출과 API 실행 권한은 점진적으로 permission 기반으로 전환한다.

## 감사 로그 정책

- 직원/권한/설정/상품/주문/결제/재고/창고/리뷰/문의 등 주요 관리자 변경 작업을 기록한다.
- 권한 부족으로 인한 403은 `PERMISSION_DENIED`로 기록한다.
- 요청 경로, 요청 메서드, IP, user agent, before/after JSON을 기록할 수 있도록 확장한다.
- 민감 정보는 원문으로 저장하지 않고 마스킹 또는 요약 기준을 따른다.
