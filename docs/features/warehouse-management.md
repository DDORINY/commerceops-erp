# 다중 창고 관리 기능 명세

| 항목 | 내용 |
| --- | --- |
| 목표 버전 | v0.6 단계 1 |
| 상태 | 단계 1 구현 완료, 통합/E2E 보강 중 |
| 작성일 | 2026-07-04 |

## 1. 목표

관리자가 여러 창고를 등록하고 상품 재고를 창고별로 배치한 뒤 창고 간 이동을 추적할 수 있게 한다.

## 2. 이번 범위

- 창고 등록과 목록 조회
- 상품 총재고 중 아직 창고에 배정되지 않은 수량의 초기 배치
- 창고별 상품 재고 조회
- 단일 상품의 창고 간 이동 요청과 완료
- 이동 완료 시 출발 창고 차감, 도착 창고 증가
- 관리자 화면과 핵심 서비스 테스트

### 제외

- 공급사·발주·입고
- 주문 재고 예약과 창고 자동 할당
- 피킹·패킹·출고
- 다품목 이동 전표와 승인 단계
- 로트·시리얼·유통기한

## 3. 재고 정합성 규칙

1. 이번 단계에서 `products.stock_quantity`는 전체 가용 재고의 기준값이다.
2. 창고별 재고 합계는 전체 재고보다 클 수 없다.
3. 초기 배치는 `전체 재고 - 창고 배정 합계` 범위 안에서만 가능하다.
4. 창고 간 이동은 전체 재고를 바꾸지 않는다.
5. 이동 완료 시 두 창고 재고를 하나의 트랜잭션과 비관적 잠금으로 변경한다.
6. 같은 창고로 이동할 수 없고 수량은 1 이상이어야 한다.
7. 완료된 이동은 다시 완료할 수 없다.

## 4. 상태 전이

| 현재 상태 | 행위 | 다음 상태 | 부수 효과 |
| --- | --- | --- | --- |
| 없음 | 이동 요청 | PENDING | 이동 전표 생성, 실제 수량 변화 없음 |
| PENDING | 이동 완료 | COMPLETED | 출발 재고 차감, 도착 재고 증가 |

## 5. API

| Method | Path | 설명 |
| --- | --- | --- |
| GET | `/api/admin/warehouses` | 창고 목록 |
| POST | `/api/admin/warehouses` | 창고 등록 |
| GET | `/api/admin/warehouse-stocks` | 창고별 재고 조회 |
| POST | `/api/admin/warehouse-stocks/allocate` | 미배정 전체 재고의 초기 배치 |
| GET | `/api/admin/stock-transfers` | 이동 목록 |
| POST | `/api/admin/stock-transfers` | 이동 요청 |
| PATCH | `/api/admin/stock-transfers/{id}/complete` | 이동 완료 |

모든 API는 `ADMIN` 또는 `SUPER_ADMIN` 권한이 필요하다.

## 6. 데이터

- `warehouses`: 창고 코드, 이름, 주소, 활성 여부
- `warehouse_stocks`: 창고/상품별 수량, 낙관적 버전
- `stock_transfers`: 출발/도착 창고, 상품, 수량, 상태, 요청/완료 시각

## 7. 수용 기준

- [x] 중복 창고 코드는 등록할 수 없다.
- [x] 전체 재고보다 많이 초기 배치할 수 없다.
- [x] 출발 재고보다 많이 이동 완료할 수 없다.
- [x] 완료 시 전체 창고 재고 합계는 변하지 않는다.
- [x] 같은 이동을 두 번 완료할 수 없다.
- [x] 백엔드 테스트와 프론트 lint/build가 통과한다.
