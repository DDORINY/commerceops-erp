# 주문 취소 및 재고 복구 명세

| 항목 | 내용 |
| --- | --- |
| 목표 버전 | v0.6 단계 3 |
| 상태 | 구현 완료, 실제 PG 환불/E2E 보강 중 |
| 작성일 | 2026-07-04 |

## 취소 가능 상태

| 주문 상태 | 취소 결과 |
| --- | --- |
| PENDING | 결제 준비 건 취소, 재고 변화 없음 |
| PAID | 결제 환불 상태, 상품 가용재고 복구, 창고 예약 해제, 회계 환불 기록 |
| PREPARING | PAID와 동일, 생성된 배송 건은 출고 전 상태 유지 |
| SHIPPING 이후 | 취소 불가, 반품 흐름 사용 |

## 원자적 처리

결제된 주문 취소는 하나의 트랜잭션에서 다음을 처리한다.

1. 창고 예약 `RESERVED → RELEASED`
2. 창고 `reserved_quantity` 감소
3. 상품 가용 총재고 증가
4. `CANCEL` 재고 로그 생성
5. 결제 상태 `REFUNDED`
6. 환불 회계 기록 생성
7. 주문 상태 `CANCELLED`, 주문 결제 상태 `REFUNDED`
8. `READY` 배송 건이 있으면 배송 상태 `CANCELLED`

어느 한 단계라도 실패하면 전체를 롤백한다. 이미 취소되거나 배송이 시작된 주문은 다시 처리하지 않는다.

## API

- 사용자: `PATCH /api/orders/{orderId}/cancel`
- 관리자: 기존 `PATCH /api/admin/orders/{orderId}/status`에 `CANCELLED` 요청

## 수용 기준

- [x] 사용자는 자신의 주문만 취소할 수 있다.
- [x] 미결제 취소는 재고와 회계를 변경하지 않는다.
- [x] 결제 취소는 상품 가용재고와 창고 예약을 정확히 복구한다.
- [x] 취소 환불 회계 기록과 재고 로그가 생성된다.
- [x] 배송 중 이후에는 취소할 수 없다.
- [x] 동일 주문 취소를 재호출해도 수량이 두 번 복구되지 않는다.
- [x] 사용자·관리자 화면에서 허용 상태만 취소할 수 있다.
