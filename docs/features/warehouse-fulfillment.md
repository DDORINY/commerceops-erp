# 창고 입고·예약·출고 연동 명세

| 항목 | 내용 |

| 작성일 | 2026-07-04 |

## 목표

상품 전체 재고와 창고 재고를 실제 주문 흐름에 연결한다. 관리자는 목적 창고에 입고/조정하고, 결제된 주문은 창고 재고를 예약하며, 배송 시작 시 출고되고 반품 승인 시 원출고 창고로 복구된다.

## 상태와 수량 규칙

```text
입고:     상품 총재고 +N / 목적 창고 실재고 +N
결제:     상품 가용 총재고 -N / 창고 예약 +N
배송 시작: 창고 실재고 -N / 창고 예약 -N / 예약 원장 SHIPPED
반품 승인: 상품 가용 총재고 +N / 원출고 창고 실재고 +N / 예약 원장 RETURNED
```

- 창고 가용수량은 `quantity - reserved_quantity`다.
- 결제 예약은 여러 창고의 재고를 순서대로 나눠 잡을 수 있다.
- 동일 주문의 예약·출고·반품 복구는 원장 상태로 멱등 처리한다.
- 입고와 재고 조정은 대상 창고를 반드시 지정한다.
- 기존 데이터는 기본 창고를 생성하고 아직 배정되지 않은 수량을 기본 창고에 배치한다.

## 데이터

- `warehouse_stocks.reserved_quantity`: 결제됐지만 아직 출고되지 않은 수량
- `stock_reservations`: 주문 품목, 창고 재고, 수량, `RESERVED/SHIPPED/RETURNED`

## 수용 기준

- [x] 지정 창고 입고 시 상품/창고 수량이 함께 증가한다.
- [x] 창고 조정 시 차이만큼 상품 총재고가 함께 변경된다.
- [x] 결제 시 창고 가용수량이 부족하면 전체 트랜잭션이 실패한다.
- [x] 예약은 같은 주문에 중복 생성되지 않는다.
- [x] 배송 시작은 예약 수량을 정확히 한 번 차감한다.
- [x] 반품 승인은 원출고 창고에 정확히 한 번 복구한다.
- [x] 테스트, lint, build가 통과한다.
