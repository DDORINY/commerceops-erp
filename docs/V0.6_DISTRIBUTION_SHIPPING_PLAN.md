# v0.6 Distribution and Shipping Plan

기준 버전: `v0.5.9-inventory-release-verification-v06-handover`

문서 상태: v0.5 재고/생산/바코드 릴리스 검증 결과를 바탕으로 정리한 v0.6 유통관리/송장/배송 계획 문서다.

## 목표

v0.6에서는 v0.5에서 마련한 SKU, 바코드, 창고 위치, 재고 실사, 안전재고 기반 위에 출고 지시, 택배사/배송 방법 관리, 송장번호 생성과 저장, 송장 라벨 출력, 배송 상태 추적 흐름을 축소 구현한다.

최종 목표는 주문 이후 운영자가 출고 준비부터 배송 완료까지 확인할 수 있는 유통관리 기반을 만들고, v0.7 회계/정산 고도화로 배송비, 환불, 정산 데이터를 넘길 수 있는 구조를 준비하는 것이다.

## v0.6 전체 범위

- 출고 주문 모델
- 주문 기반 출고 지시 생성
- 출고 품목과 SKU/바코드 연결
- 출고 검수 상태 관리
- 택배사/배송 방법 관리
- 송장번호 생성 또는 수동 입력
- 송장 라벨 출력 후보
- 배송 상태 기록
- 배송 추적 이벤트 저장
- 반품/교환 배송 흐름 후보
- 바코드 기반 출고 검수 후보
- v0.7 회계/정산으로 넘길 배송비/환불 데이터 후보 정리

## DB 설계 후보

| 테이블 | 목적 | 핵심 필드 후보 |
| --- | --- | --- |
| `outbound_orders` | 출고 지시 header | `id`, `outbound_number`, `order_id`, `warehouse_id`, `status`, `requested_at`, `picked_at`, `shipped_at`, `memo`, timestamps |
| `outbound_order_items` | 출고 품목 | `id`, `outbound_order_id`, `order_item_id`, `sku_id`, `product_id`, `quantity`, `picked_quantity`, `scanned_quantity`, timestamps |
| `carriers` | 택배사 관리 | `id`, `name`, `code`, `tracking_url_template`, `active`, timestamps |
| `shipping_methods` | 배송 방법/정책 | `id`, `carrier_id`, `name`, `code`, `base_fee`, `active`, timestamps |
| `shipment_labels` | 송장 라벨/출력 이력 | `id`, `shipment_id`, `tracking_number`, `label_format`, `print_count`, `last_printed_at`, `created_by`, timestamps |
| `shipment_tracking_events` | 배송 추적 이벤트 | `id`, `shipment_id`, `status`, `description`, `event_at`, `raw_payload`, timestamps |
