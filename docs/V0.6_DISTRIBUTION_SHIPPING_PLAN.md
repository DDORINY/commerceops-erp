# v0.6 유통관리/송장/배송 계획

기준 버전: `v0.6.0-distribution-shipping-plan`

문서 상태: v0.6.0에서 확정하는 계획 문서다. v0.6.1부터 기능 구현을 시작하며, 이 문서의 범위를 기준으로 체크리스트와 구현 범위를 관리한다.

## 목표

v0.6에서는 v0.5에서 구현한 SKU, 바코드, 생산 입고, 재고 실사, 창고 위치, 안전재고 기반 위에 실무 ERP의 출고/배송 흐름을 축소 구현한다.

핵심 목표는 주문 이후 운영자가 출고 지시를 만들고, 바코드로 출고를 검수하며, 택배사와 배송 방법을 선택하고, 송장번호와 송장 라벨을 관리하며, 배송 상태와 추적 이벤트를 기록할 수 있는 운영 기반을 만드는 것이다.

v0.6에서 생성되는 배송비, 환불 배송비, 출고 완료 기준 매출 확정 후보 데이터는 v0.7 회계/정산 고도화로 넘긴다.

## 기존 구조 기준

- 주문: `Order`, `OrderItem`, `OrderStatus`로 주문과 주문 품목을 관리한다.
- 배송: `Shipment`는 현재 주문 1:1 구조로 `trackingNumber`, `carrier`, `ShipmentStatus`, `shippedAt`, `deliveredAt`을 보유한다.
- 재고: `WarehouseStock`, `WarehouseLocationStock`, `InventoryLog`가 상품/창고/위치 단위 재고 변경의 기반이다.
- SKU/바코드: `Sku`는 상품과 연결되며 `skuCode`, `barcode`, `safetyStockQuantity`, `active`를 보유한다.
- 권한: role 기반 1차 관리자 접근과 permission code 기반 세부 실행 권한을 병행한다.
- 감사 로그: `AuditLogService`로 주요 관리자 변경 작업을 기록한다.
- 관리자 메뉴: `AdminSidebarV2`와 `adminMenu.ts`는 permission 기반 메뉴 노출을 지원한다.

## v0.5에서 이관된 이슈

| `RETURN_SHIPPING_MANAGE` | 반품/교환 배송 관리 |

SUPER_ADMIN은 전체 권한을 보유한다. 기존 role 기반 1차 관리자 접근은 유지하고, 세부 실행은 permission code 기준으로 점진 전환한다.
