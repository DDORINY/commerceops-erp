# 버전별 구현 계획과 체크리스트

## 사용 방법

- `[ ]` 예정, `[x]` 완료로 관리한다.
- 코드가 있어도 공통 완료 조건을 통과하기 전에는 체크하지 않는다.
- 각 버전을 시작할 때 담당자, 목표일, 관련 기능 명세 링크를 추가한다.
- 버전 종료 시 [릴리스 체크리스트](./templates/RELEASE_CHECKLIST.md)를 복사해 결과를 남긴다.

## 공통 완료 조건(Definition of Done)

모든 기능은 아래 조건을 충족해야 완료다.

- [ ] 사용자 시나리오와 예외 시나리오가 기능 명세에 정의됨
- [ ] 권한, 입력 검증, 상태 전이 규칙이 구현됨
- [ ] DB 변경이 마이그레이션으로 관리되고 롤백/호환성을 검토함
- [ ] API 명세와 실제 요청·응답이 일치함
- [ ] 백엔드 단위/통합 테스트와 핵심 프론트 테스트가 있음
- [ ] 백엔드 테스트, 프론트 lint/build가 통과함
- [ ] 로딩·빈 상태·오류·재시도 UI를 처리함
- [ ] 로그에 비밀번호·토큰·불필요한 개인정보가 남지 않음
- [ ] 관련 API/DB/운영 문서를 함께 갱신함
- [ ] 수동 인수 테스트 결과를 기록함
