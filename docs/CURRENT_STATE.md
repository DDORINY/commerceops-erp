# 현재 구현 현황

기준 버전: `v0.2.9`
기준일: 2026-07-05

이 문서는 v0.1.1 ~ v0.1.9 완료 상태, v0.2.0 계획 수립 기준, v0.2.1 인증/세션/보안 안정화, v0.2.2 결제/환불, v0.2.3 미디어 업로드, v0.2.4 리뷰 운영/감사 로그, v0.2.5 DB 마이그레이션/스키마 정리, v0.2.6 CI/환경 분리, v0.2.7 알림/관리자 권한, v0.2.8 운영 분석 기반, v0.2.9 릴리스 검증 결과를 함께 기록한다.

## 요약

CommerceOps ERP는 Spring Boot 백엔드와 Next.js 프론트엔드로 구성된 쇼핑몰/관리자 ERP 프로젝트다. 현재 MVP 이후 v0.1.1 ~ v0.1.8 작업을 통해 주요 관리자 운영 화면과 사용자 쇼핑 화면의 mock 데이터를 제거하거나 실제 API 흐름으로 정리했고, v0.1.9에서 계획표/체크리스트/태그/권한/mock 제거 상태를 최종 검증했다. v0.2.0부터는 결제, 인증/보안, DB/배포, 미디어, 운영 고도화 계획을 버전 단위로 진행하며, v0.2.1에서 로그아웃/리프레시 토큰/세션 만료 UX를 최소 안정 범위로 구현했고, v0.2.2에서 결제 승인/취소 API 경계와 멱등성 키를 정리했다. v0.2.3에서는 관리자 상품 이미지 업로드와 미디어 메타데이터 관리를 추가했고, v0.2.4에서는 리뷰 숨김/해제와 리뷰 운영 감사 로그를 추가했다. v0.2.5에서는 Flyway 기반 초기 DDL과 DB 마이그레이션 운영 기준을 추가했고, v0.2.6에서는 GitHub Actions CI와 운영 환경 변수 분리 기준을 추가했다. v0.2.7에서는 주문/문의/반품 알림과 MANAGER 조회 권한 정책을 추가했다. v0.2.8에서는 회계/매출/창고 데이터를 읽기 전용으로 모으는 운영 분석 기초 API와 프론트 service 타입을 추가했다. v0.2.9에서는 v0.2 체크리스트/태그/mock/JWT 권한 흐름을 검증하고, v0.3 이후 AI 커머스 ERP 장기 로드맵을 문서화했다.

## 검증 기준 이력

최근 각 버전 작업에서 아래 검증을 실행했다.

- frontend `npm.cmd run lint`
- frontend `npm.cmd run build`
- backend `.\gradlew.bat test`

Windows 환경에서 기본 `.next` 디렉터리 EPERM 잠금이 발생하는 경우 `NEXT_DIST_DIR=.next-build-check-v0xx` 형식의 별도 빌드 디렉터리로 검증하고, 완료 후 임시 산출물을 삭제한다.

v0.1.9에서는 아래 항목을 릴리스 검증 기준으로 추가 확인한다.

- `v0.1.1` ~ `v0.1.8` 체크리스트 파일 존재.
- `v0.1.1` ~ `v0.1.8` 태그 존재.
- 관리자/사용자 화면의 기존 mock 데이터 import 제거.
- `SecurityConfig`와 프론트 `apiClient`의 JWT Bearer/401/403 처리 흐름.
- 주문/상품/재고/배송/반품/쿠폰/문의/리뷰/회계/창고 주요 흐름의 문서 기준 확인 항목.
