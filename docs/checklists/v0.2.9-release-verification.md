## mock 제거 확인

- 관리자 화면: `mockProducts`, `mockOrders`, `mockUsers`, `dummy`, `sample`, `hardcoded` 패턴 검색 결과 없음.
- 사용자 화면: `frontend/src/features/*/mock.ts` 파일 없음.
- 예외: `POST /api/payments/mock/complete`와 `paymentService.mockComplete`는 v0.2.2에서 `/api/payments/approve`로 위임하는 하위 호환 API로 유지한다.

## JWT/권한 확인

- `apiClient`는 `Authorization: Bearer {accessToken}` 헤더를 추가한다.
- 401 발생 시 refresh token으로 1회 재발급을 시도하고, 실패하면 토큰과 사용자 상태를 제거한 뒤 로그인 페이지로 이동한다.
- `SecurityConfig`는 공개 API, 사용자 인증 API, 관리자 API 권한을 구분한다.
- `MANAGER`는 운영 조회 중심 GET API 접근이 가능하고, 변경/회계/쿠폰/감사/리뷰 운영 등은 `ADMIN`, `SUPER_ADMIN` 중심으로 제한된다.
