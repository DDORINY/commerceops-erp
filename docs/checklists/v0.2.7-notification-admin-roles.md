## 구현 결정

- 실제 이메일/SMS 발송은 이번 범위에서 제외하고 DB 기반 in-app 알림 API를 먼저 둔다.
- 알림 이벤트는 주문 상태 변경, 문의 답변, 반품 승인/거절부터 기록한다.
- `MANAGER`는 운영 조회 중심 GET API에 접근할 수 있다.
- 사용자/권한 변경, 회계, 쿠폰, 상품 변경, 리뷰 운영, 감사 로그는 `ADMIN`, `SUPER_ADMIN`으로 유지한다.

## 검증 결과

- frontend `npm.cmd run lint`: 통과
- frontend `npm.cmd run build`: 기본 `.next` EPERM 발생, `NEXT_DIST_DIR=.next-build-check-v027` 재실행 중 EPERM 발생, 권한 상승 재실행으로 통과
- backend `.\gradlew.bat test`: Gradle wrapper lock 권한 오류 후 권한 상승 재실행. 새 `NotificationService` 의존성으로 기존 단위 테스트 NPE 발생, 테스트 mock 보강 후 통과
- 임시 산출물 `.next-build-check-v027`: 삭제 완료
- `frontend/tsconfig.json`: Next가 추가한 임시 include 제거 완료
