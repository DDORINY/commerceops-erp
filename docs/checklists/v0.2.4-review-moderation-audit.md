## 구현 결정

- 리뷰 상태는 `VISIBLE`, `HIDDEN`, `DELETED` enum으로 관리한다.
- 기존 데이터 호환을 위해 DB의 null 상태는 응답/쿼리에서 `VISIBLE`로 취급한다.
- 사용자 상품 상세 리뷰 목록은 `VISIBLE` 리뷰만 조회한다.
- 관리자 리뷰 목록은 `DELETED`가 아닌 리뷰를 조회하므로 숨김 리뷰도 확인할 수 있다.
- 사용자/관리자 삭제 API는 물리 삭제 대신 `DELETED` 상태로 바꾸는 soft delete로 처리한다.
- 숨김은 운영상 노출 제어이고, 삭제는 위험 작업으로 문서와 UI에서 분리한다.
- 감사 로그는 리뷰 숨김/해제/삭제 작업부터 기록하며, 다른 관리자 기능으로 확장 가능한 actor/target/action 구조로 만들었다.
