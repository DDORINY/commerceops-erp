## 구현 요약

- `tracking_number_source`, `tracking_number_issued_at` 컬럼을 `shipments`에 추가했다.
- `POST /api/admin/shipments/{shipmentId}/tracking-number`는 `TRK-yyyyMMdd-000001` 형식의 내부 테스트용 송장번호를 생성한다.
- `PATCH /api/admin/shipments/{shipmentId}/tracking-number`와 기존 `PATCH /api/admin/shipments/{shipmentId}/tracking`는 수동 송장번호 저장/수정 흐름을 유지한다.
- 송장번호 저장/자동 생성 시 `TrackingNumberSource.MANUAL` 또는 `TrackingNumberSource.SYSTEM`을 기록한다.
- 최초 READY 배송 건에 송장을 등록하면 기존 배송 시작 처리와 동일하게 예약 재고 출고와 주문 `SHIPPING` 전환을 수행한다.
- 이미 IN_TRANSIT인 배송 건의 송장번호 수정은 재고 차감을 반복하지 않는다.

## 권한과 감사 로그

- 배송 조회: `SHIPMENT_READ`
- 송장번호 생성/수정/배송완료: `SHIPMENT_MANAGE`
- 감사 로그:
  - `TRACKING_NUMBER_GENERATED`
  - `TRACKING_NUMBER_UPDATED`
