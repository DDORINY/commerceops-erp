- [x] `SUPER_ADMIN`은 전체 권한 허용
- [x] `ADMIN`은 운영 권한 중심
- [x] `MANAGER`는 조회 중심
- [x] `USER`와 비로그인은 관리자 접근 불가
- [x] 기존 role은 `/api/admin/**` 1차 접근 기준으로 유지
- [x] permission code는 세부 실행 권한 기준으로 적용
- [x] `AdminSidebarV2`는 effective permission 기반 메뉴 노출
- [x] 주요 관리자 API는 `PermissionChecker` 기준 권한 검증
- [x] 권한 실패 시 403 한국어 메시지 반환
- [x] 권한 실패는 `PERMISSION_DENIED` audit log 기록
