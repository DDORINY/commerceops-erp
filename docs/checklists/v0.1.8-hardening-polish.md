## 결과

- 변경 파일:
  - `frontend/src/lib/api.ts`
  - `frontend/src/app/login/page.tsx`
  - `frontend/src/app/products/[id]/page.tsx`
  - `frontend/src/app/cart/page.tsx`
  - `frontend/src/app/wishlist/page.tsx`
  - `frontend/src/app/orders/checkout/page.tsx`
  - `docs/CURRENT_STATE.md`
  - `docs/checklists/v0.1.8-hardening-polish.md`
- 안정화 결과:
  - `apiClient`에 네트워크 실패 방어 메시지를 추가했다.
  - 401 발생 시 로그인 페이지로 이동하며 세션 만료/로그인 필요 메시지를 표시한다.
  - 403 권한 오류 메시지를 사용자에게 더 명확하게 표시한다.
  - 상품 상세 위시리스트 상태 조회가 실제 `accessToken` 키를 사용하도록 수정했다.
  - 상품 상세 리뷰/문의 탭 API 실패 시 화면 내 메시지를 표시한다.
  - 장바구니, 위시리스트, 체크아웃, 상품 상세의 주요 액션 alert 일부를 화면 내 메시지로 전환했다.
- 검증 결과:
  - frontend `npm.cmd run lint`: 통과
  - frontend `npm.cmd run build`: 기본 `.next` EPERM 발생 후 `NEXT_DIST_DIR=.next-build-check-v018`로 재검증 통과
  - backend `.\gradlew.bat test`: 최초 Gradle wrapper lock 권한 오류 후 권한 밖 재실행 통과
  - `.next-build-check-v018` 임시 산출물 삭제 완료
  - `tsconfig.json` 임시 include 원복 확인
- 남은 이슈:
  - 관리자 일부 처리성 액션은 여전히 `alert`/`confirm`을 사용한다. 대형 UX 컴포넌트 추가 없이 v0.1.8에서는 사용자 핵심 구매 흐름과 공통 API 실패 흐름을 우선 보강했다.
  - 실제 PG, 이미지 업로드, 고급 BI/WMS/복식부기는 v0.2 이후 범위로 유지한다.
