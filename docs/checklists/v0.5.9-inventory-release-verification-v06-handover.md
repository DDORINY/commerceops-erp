## v0.5 검증 요약

- `v0.5.0`: 재고/생산/바코드 자동화 계획을 수립했다.
- `v0.5.1`: SKU/바코드 마스터, 관리자 SKU API/UI, SKU/바코드 권한을 추가했다.
- `v0.5.2`: 생산 주문, 생산 완료 입고, 상품/창고 재고 증가 흐름을 추가했다.
- `v0.5.3`: 바코드 라벨 HTML 미리보기와 출력 이력 기록 기반을 추가했다.
- `v0.5.4`: 바코드 기준 SKU/상품/창고 재고 조회와 간단 입고/출고 처리를 추가했다.
- `v0.5.5`: 창고별 재고 실사 세션, 품목 snapshot, 차이 조정 흐름을 추가했다.
- `v0.5.6`: 창고 내부 위치/로케이션과 위치별 SKU 재고 조회 기반을 추가했다.
- `v0.5.7`: SKU 또는 SKU+창고 단위 안전재고 기준과 재고 부족 조회 화면을 추가했다.
- `v0.5.8`: 재고/생산/바코드 관리자 화면 파일과 문서 정합성을 검증했다.

## 남은 이슈와 v0.6 이관

- 실제 바코드 프린터 SDK/PDF 라벨 출력은 v0.6 이후 배송 라벨 출력과 함께 고도화한다.
- 재고 변경 통합 ledger와 출고/배송 흐름의 연결 정책은 v0.6 출고 모델에서 재검토한다.
- 위치별 SKU 재고와 상품/창고 재고 간 source of truth 정책은 출고 검수와 함께 보강한다.
- 자동 발주, AI 수요 예측, 외부 알림 채널 연동은 v0.7 이후 또는 AI 단계로 이관한다.
- 실제 브라우저 기반 수동 확인은 로컬 서버 실행 상태에서 별도 운영 점검으로 진행한다.
