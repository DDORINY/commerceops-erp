## 구현 결정

- 기존 상품 대표 이미지는 `products.image_url` 필드를 유지한다.
- 업로드 API는 `MediaFileResponse.url`을 반환하고, 관리자 폼은 이 URL을 `imageUrl`에 자동 입력한다.
- URL 직접 입력 방식은 외부 이미지나 기존 운영 데이터를 위해 유지한다.
- 업로드 파일은 기본 `uploads/product-images/{yyyy-MM-dd}` 하위에 저장된다.
- 공개 URL의 기본 base는 `http://localhost:8080`, 설정값은 `COMMERCEOPS_MEDIA_PUBLIC_BASE_URL`로 분리했다.
- 운영용 S3/CDN, 리사이징, 썸네일, 다중 이미지 갤러리는 v0.2 이후 고도화 범위로 남긴다.

## 보안 기준

- 허용 MIME 타입: `image/jpeg`, `image/png`, `image/webp`, `image/gif`.
- 허용 확장자: `jpg`, `jpeg`, `png`, `webp`, `gif`.
- 최대 파일 크기: 5MB.
- 원본 파일명은 저장 파일명으로 사용하지 않고 UUID 저장 파일명을 사용한다.
- `..`, `/`, `\`가 포함된 파일명과 업로드 경로 이탈은 거부한다.
- 프론트에는 로컬 파일 시스템 절대 경로를 노출하지 않고 공개 URL만 반환한다.
