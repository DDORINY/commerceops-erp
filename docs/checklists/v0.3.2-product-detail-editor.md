# v0.3.2-product-detail-editor 체크리스트

## 목표

상품 상세페이지를 단일 설명 필드 중심에서 블록 기반 CMS 구조로 확장한다. 관리자는 상품별 상세 블록을 추가/수정/삭제/정렬/노출 제어할 수 있고, 사용자는 `visible=true` 블록만 `sortOrder` 기준으로 조회한다.

## 구현 요약

- 관리자 API:
  - `GET /api/admin/products/{productId}/detail-blocks`
  - `PUT /api/admin/products/{productId}/detail-blocks`
- 사용자 API:
  - `GET /api/products/{productId}` 응답에 `detailBlocks` 포함
- 블록 타입:
  - `HEADING`, `TEXT`, `IMAGE`, `NOTICE`, `SPEC_TABLE`, `HTML`
- 저장 방식:
  - PUT 요청 배열을 productId 기준으로 전체 교체 저장
  - 서버가 요청 순서대로 `sortOrder`를 재보정
  - `visible=false` 블록은 사용자 응답에서 제외

## 남은 이슈

- 드래그앤드롭 고급 편집기는 제외 범위로 유지.
- HTML 블록은 관리자 내부 CMS 입력을 신뢰하는 최소 구현이다. 운영 환경에서는 sanitizer 도입 또는 허용 태그 정책이 필요하다.
- 이미지 블록은 URL 입력 기반이며, 상세 블록 내부 직접 파일 업로드 UX는 후속 개선 후보.
