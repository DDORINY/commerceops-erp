# v0.2.1.1 관리자 DataTable key 중복 hotfix 체크리스트

- 기준 버전: `v0.2.1`
- 패치 태그: `v0.2.1.1`
- 브랜치: `v0.2.1-hotfix-admin-orders-key`
- 작업명: 관리자 DataTable 액션 컬럼 key 중복 수정
- 기준일: 2026-07-05

## 문제 증상

- `/admin/orders` 접속 시 React 콘솔에 `Encountered two children with the same key, orderId` 경고가 발생했다.
- `DataTable`이 `columns[].key`를 `th`, `td`의 React key로 사용하는데, ID 컬럼과 관리 컬럼이 같은 `orderId`를 사용했다.

## 수정 내용

- `/admin/orders` 관리 컬럼 key를 `orderId`에서 `actions`로 변경.
- 추가 점검 중 같은 패턴을 가진 `/admin/inventory`, `/admin/products` 관리 컬럼도 `actions`로 변경.
- ID 컬럼 key는 유지하고, 관리 컬럼 render와 상태 변경/수정/삭제 동작은 변경하지 않았다.

## 점검한 관리자 DataTable 페이지

- [x] `/admin/orders`: `orderId` 중복 수정
- [x] `/admin/inventory`: `productId` 중복 수정
- [x] `/admin/products`: `id` 중복 수정
- [x] `/admin`, `/admin/accounting`, `/admin/coupons`, `/admin/customers`, `/admin/returns`, `/admin/shipments`, `/admin/reviews`, `/admin/sales`, `/admin/warehouses`: 검색 기준 중복 액션 key 없음 확인

## 검증 결과

- frontend `npm.cmd run lint`: 통과
- frontend `npm.cmd run build`: 기본 `.next`에서 EPERM 발생 후 `NEXT_DIST_DIR=.next-build-check-key-fix`로 재실행해 통과
- 임시 빌드 산출물: `.next-build-check-key-fix` 삭제 완료
- `tsconfig.json` 임시 변경: Next가 추가한 `.next-build-check-key-fix` include 제거 완료, 최종 변경 없음
