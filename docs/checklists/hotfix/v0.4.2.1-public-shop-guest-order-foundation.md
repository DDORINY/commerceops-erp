## 비회원 주문 설계

후속 주문 버전에서 아래 구조로 구현한다.

- `orderType`: `MEMBER` / `GUEST`
- `userId`: 회원 주문이면 값 있음, 비회원 주문이면 `null`
- `guestName`
- `guestPhone`
- `guestEmail`
- `guestOrderPasswordHash`
- `guestOrderLookupToken` 또는 주문번호 기반 조회

비회원 주문 흐름:
1. 비회원 장바구니 또는 바로구매
2. 주문자 정보 입력
3. 배송지 정보 입력
4. 비회원 주문 비밀번호 입력
5. 주문 생성
6. 결제
7. 주문번호 발급
8. 주문번호 + 전화번호/비밀번호로 조회
