## 구현 결정

- v0.2.2는 실제 PG 벤더 키/웹훅/리다이렉트 연동까지 구현하지 않는다.
- 외부 PG 연동을 위한 API 경계를 먼저 확정하고, 현재는 `MOCK_PROVIDER` 기반 승인으로 동작한다.
- `POST /api/payments/approve`는 `orderId`, `paymentMethod`, `providerTransactionId`, `idempotencyKey`를 받는다.
- `idempotencyKey`가 같은 승인 요청은 기존 결제 응답을 반환해 중복 재고 차감과 중복 회계 매출 기록을 방지한다.
- `POST /api/payments/{paymentId}/cancel`은 사용자 소유 결제만 처리하며, 기존 `OrderCancellationService`를 재사용한다.
- 환불은 결제 상태 `REFUNDED`, 주문 결제 상태 `REFUNDED`, 회계 환불 기록, 상품/창고 재고 복구를 하나의 트랜잭션으로 처리한다.
- 기존 `/api/payments/mock/complete`는 `/api/payments/approve`로 위임하는 하위 호환 API로 유지한다.
