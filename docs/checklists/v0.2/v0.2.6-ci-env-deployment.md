## 구현 결정

- CI는 GitHub Actions로 구성한다.
- frontend는 Node 20과 `npm ci`를 사용한다.
- backend는 Temurin Java 17과 Gradle cache를 사용한다.
- 실제 배포 자동화는 이번 범위에서 제외하고, lint/build/test 자동 검증과 환경 분리 문서화를 우선한다.
- 운영 backend profile은 `application-prod.yml`로 추가하고, DB/JWT secret은 환경 변수로만 주입한다.
- CORS origin은 `COMMERCEOPS_CORS_ALLOWED_ORIGINS`로 분리한다.

## 검증 결과

- frontend `npm.cmd run lint`: 통과
- frontend `npm.cmd run build`: 기본 `.next` EPERM 발생, `NEXT_DIST_DIR=.next-build-check-v026` 재실행 중 EPERM 발생, 권한 상승 재실행으로 통과
- backend `.\gradlew.bat test`: Gradle wrapper lock 권한 오류 발생, 권한 상승 재실행으로 통과
- 임시 산출물 `.next-build-check-v026`: 삭제 완료
- `frontend/tsconfig.json`: Next가 추가한 임시 include 제거 완료
