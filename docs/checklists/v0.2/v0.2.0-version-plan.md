## v0.2 계획 원칙

- v0.2는 v0.1에서 연결된 실제 API 기반 운영 흐름을 제품 수준으로 고도화한다.
- 결제/보안/배포/DB처럼 운영 리스크가 큰 항목을 우선한다.
- 각 버전은 하나의 명확한 목표만 다룬다.
- 고급 기능은 최소 운영 가능한 범위부터 구현하고, 대규모 확장은 v0.3 이후로 넘긴다.
- 모든 개발 버전은 체크리스트, lint/build/test, 문서 갱신, main 병합, 태그 생성까지 완료한다.

## 검증 결과

- frontend `npm.cmd run lint`: 통과
- frontend `npm.cmd run build`: 기본 `.next`에서 EPERM 발생 후 `NEXT_DIST_DIR=.next-build-check-v020`로 재실행해 통과
- backend `.\gradlew.bat test`: 일반 권한에서 Gradle lock 접근 오류 발생 후 권한 승격으로 재실행해 통과
- 임시 빌드 산출물: `.next-build-check-v020` 삭제 완료
- `tsconfig.json` 임시 변경: Next가 추가한 `.next-build-check-v020` include 제거 완료, 최종 변경 없음
