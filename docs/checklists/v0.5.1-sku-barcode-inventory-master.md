## SKU/바코드 정책

- `Product.productCode`는 상품 마스터 코드로 유지한다.
- `Sku.skuCode`는 재고, 입출고, 바코드, 생산 입고에서 사용할 운영 단위 코드다.
- SKU 코드는 수동 입력이 가능하며 비어 있으면 `productCode-001` 또는 `SKU-{productId}-001` 형식으로 자동 생성한다.
- 바코드는 수동 입력이 가능하며 비어 있으면 `BC{productId}{sequence}` 형식의 고유 문자열로 자동 생성한다.
- SKU 코드와 바코드는 각각 중복을 허용하지 않는다.
- 바코드 재발급은 기존 바코드를 새 고유 값으로 교체하고 감사 로그를 남긴다.

## 권한 기준

- SKU 조회: `INVENTORY_READ`
- SKU 생성/수정/활성 변경: `SKU_MANAGE`
- 바코드 재발급: `BARCODE_MANAGE`
- `SUPER_ADMIN`은 기존 정책에 따라 모든 권한을 보유한다.
- `ADMIN_GROUP`에는 `SKU_MANAGE`, `BARCODE_MANAGE`를 seed한다.
- `MANAGER_GROUP`은 조회 중심 정책을 유지하며 `INVENTORY_READ` 기반으로 목록 접근만 허용한다.
