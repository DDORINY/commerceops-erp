  - `docs/operations/CURRENT_STATE.md`
  - `docs/checklists/v0.1.5-admin-warehouses-api.md`
- API 엔드포인트:
  - `GET /api/admin/warehouses`
  - `POST /api/admin/warehouses`
  - `GET /api/admin/warehouse-stocks?warehouseId={warehouseId}&keyword={keyword}&page={page}&size={size}`
  - `POST /api/admin/warehouse-stocks/allocate`
  - `GET /api/admin/stock-transfers?status={status}&page={page}&size={size}`
  - `POST /api/admin/stock-transfers`
  - `PATCH /api/admin/stock-transfers/{id}/complete`
- mock 제거 여부: 관리자 창고 관리 화면과 warehouse service에는 mock 데이터가 없고, 실제 API만 사용한다.
- 검증 결과:
  - frontend `npm.cmd run lint`: 통과
  - frontend `npm.cmd run build`: 기본 `.next` EPERM 발생 후 `NEXT_DIST_DIR=.next-build-check-v015`로 재검증 통과
  - backend `.\gradlew.bat test`: 통과
- 남은 이슈:
  - 창고 비활성화/삭제 정책은 아직 없다.
  - 피킹/패킹/출고 자동화와 복잡한 WMS 정책은 v0.2 이후 고도화 범위로 유지한다.
