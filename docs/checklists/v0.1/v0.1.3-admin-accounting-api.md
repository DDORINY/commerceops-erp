## 결과

- 변경 파일:
  - `frontend/src/app/admin/accounting/page.tsx`
  - `frontend/src/lib/services/accountingService.ts`
  - `frontend/src/lib/format.ts`
  - `docs/architecture/API_REFERENCE.md`
  - `docs/operations/CURRENT_STATE.md`
  - `docs/checklists/v0.1.3-admin-accounting-api.md`
- API 엔드포인트:
  - `GET /api/admin/accounting/summary`
  - `GET /api/admin/accounting/entries?type={type}&page={page}&size={size}`
- mock 제거 여부: 관리자 회계 화면과 회계 service에는 mock 데이터가 없고, 실제 API만 사용한다.
- CSV 기준: 서버 CSV API 없이 현재 조회된 회계 내역 페이지 기준으로 `ID`, `구분`, `금액`, `설명`, `참조ID`, `일시` 컬럼을 클라이언트에서 생성한다.
- 검증 결과:
  - frontend `npm.cmd run lint`: 통과
  - frontend `npm.cmd run build`: 기본 `.next` EPERM 발생 후 `NEXT_DIST_DIR=.next-build-check-v013`로 재검증 통과
  - backend `.\gradlew.bat test`: 통과
- 남은 이슈:
  - 현재 회계 도메인은 `SALE`, `REFUND`, `INBOUND` 단식 이벤트 기록이다.
  - 복식부기, 계정과목, 마감, 정산 보고서, 서버 CSV 다운로드 API는 v0.2 이후 고도화 범위로 유지한다.
