## 결과

- 변경 파일:
  - `docs/architecture/API_REFERENCE.md`
  - `docs/architecture/DATABASE_SCHEMA.md`
  - `docs/architecture/FRONTEND_STRUCTURE.md`
  - `docs/architecture/BACKEND_STRUCTURE.md`
  - `docs/operations/CURRENT_STATE.md`
  - `docs/checklists/v0.1.7-docs-api-db-sync.md`
- 문서 동기화 결과:
  - 실제 Controller 기준 API 경로, method, 권한, request/response DTO를 정리했다.
  - 실제 Entity 기준 논리 DB 스키마로 정리하고, DDL 파일 부재를 명시했다.
  - 실제 Next app route와 service 파일 기준으로 프론트 구조를 정리했다.
  - 실제 백엔드 domain/controller/service/repository/entity 기준으로 백엔드 구조를 정리했다.
  - 실제 PG, 리뷰 숨김, 파일 업로드, 고급 BI/WMS/회계 기능은 미구현/예정으로 분리했다.
- 검증 결과:
  - frontend `npm.cmd run lint`: 통과
  - frontend `npm.cmd run build`: 기본 `.next` EPERM 발생 후 `NEXT_DIST_DIR=.next-build-check-v017`로 재검증 통과
  - backend `.\gradlew.bat test`: 최초 Gradle wrapper lock 권한 오류 후 권한 밖 재실행 통과
  - `.next-build-check-v017` 임시 산출물 삭제 완료
  - `tsconfig.json` 임시 include 원복 확인
- 남은 이슈:
  - 실제 MySQL DDL 마이그레이션 파일이 없어 DB 문서는 엔티티 기준 논리 스키마로 유지한다.
  - 운영 DB 인덱스/외래키명/제약명은 추후 마이그레이션 도입 시 확정해야 한다.
  - 미구현 기능은 `CURRENT_STATE.md`의 미구현/예정 항목으로 분리했다.
