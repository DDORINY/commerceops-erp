## 결과

- 변경 파일:
  - `frontend/src/app/admin/sales/page.tsx`
  - `frontend/src/lib/services/adminService.ts`
  - `docs/architecture/API_REFERENCE.md`
  - `docs/operations/CURRENT_STATE.md`
  - `docs/checklists/v0.1.4-admin-sales-api.md`
- API 엔드포인트:
  - `GET /api/admin/dashboard/summary`
  - `GET /api/admin/dashboard/sales?period={period}&startDate={startDate}&endDate={endDate}`
  - `GET /api/admin/dashboard/top-products?limit={limit}`
  - `GET /api/admin/orders?status=PAID&page={page}&size={size}`
- mock 제거 여부: 관리자 매출 통계 화면과 admin service에는 mock 데이터가 없고, 실제 API만 사용한다.
- 검증 결과:
  - frontend `npm.cmd run lint`: 통과
  - frontend `npm.cmd run build`: 기본 `.next` EPERM 발생 후 `NEXT_DIST_DIR=.next-build-check-v014`로 재검증 통과
  - backend `.\gradlew.bat test`: 통과
- 남은 이슈:
  - 인기 상품 API는 전체 주문 상품 기준이며 기간 필터를 받지 않는다.
  - 기간별 상품 성과, 고급 BI, 차트 라이브러리 교체, 정산/세무 리포트는 v0.2 이후 고도화 범위로 유지한다.
