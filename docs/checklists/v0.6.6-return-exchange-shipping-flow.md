## 구현 내용

- 반품 배송 정보는 `return_shipment_infos` 테이블에 반품 요청과 1:1로 저장한다.
- 상태 값은 `NOT_REQUESTED`, `COLLECTION_REQUESTED`, `IN_TRANSIT`, `RECEIVED`, `CANCELLED`를 사용한다.
- 배송비 부담 주체는 `UNDECIDED`, `CUSTOMER`, `COMPANY`를 사용한다.
- 관리자 API는 반품 배송 조회에는 `ORDER_READ`, 저장에는 `RETURN_SHIPPING_MANAGE` 권한을 요구한다.
- 실제 택배사 회수 API 연동, 교환 재출고 자동화, 환불 정산 마감 자동화는 후속 범위로 유지한다.
