## API

- `GET /api/admin/carriers`: 택배사 목록 조회, `CARRIER_MANAGE`
- `POST /api/admin/carriers`: 택배사 생성, `CARRIER_MANAGE`
- `PATCH /api/admin/carriers/{carrierId}`: 택배사 수정, `CARRIER_MANAGE`
- `PATCH /api/admin/carriers/{carrierId}/active`: 택배사 활성 상태 변경, `CARRIER_MANAGE`
- `GET /api/admin/shipping-methods`: 배송 방법 목록 조회, `CARRIER_MANAGE`
- `POST /api/admin/shipping-methods`: 배송 방법 생성, `CARRIER_MANAGE`
- `PATCH /api/admin/shipping-methods/{shippingMethodId}`: 배송 방법 수정, `CARRIER_MANAGE`
- `PATCH /api/admin/shipping-methods/{shippingMethodId}/active`: 배송 방법 활성 상태 변경, `CARRIER_MANAGE`

## 정책

- 택배사 코드는 unique이며 서버에서 대문자로 정규화한다.
- 배송 방법 코드는 unique이며 서버에서 대문자로 정규화한다.
- 배송 방법은 택배사와 선택적으로 연결할 수 있다.
- 비활성 택배사는 신규 배송 방법에 연결할 수 없다.
- 기본 배송비는 0원 이상으로 제한한다.
- 실제 택배사 API 호출, 자동 운임 계산, 지역별 배송비 매트릭스는 후속 범위다.
