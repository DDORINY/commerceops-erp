## 구현 요약

- `shipment_tracking_events`는 배송 건, 상태, 설명, 이벤트 시각, 원시 payload 후보를 저장한다.
- 관리자 배송 화면에서 배송 건별 추적 이력을 조회하고, 수동 이벤트를 추가할 수 있다.
- 관리자 배송 화면에서 배송 상태를 `READY`, `IN_TRANSIT`, `DELIVERED`, `CANCELLED` 중 하나로 변경할 수 있다.
- 상태 변경 시 추적 이벤트와 audit log를 함께 남긴다.

## API

- `GET /api/admin/shipments/{shipmentId}/tracking-events`: 배송 추적 이력 조회, `SHIPMENT_READ`
- `POST /api/admin/shipments/{shipmentId}/tracking-events`: 배송 추적 이벤트 추가, `SHIPMENT_MANAGE`
- `PATCH /api/admin/shipments/{shipmentId}/status`: 배송 상태 변경, `SHIPMENT_MANAGE`
