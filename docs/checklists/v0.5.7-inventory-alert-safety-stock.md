## 구현 내용

- `inventory_alert_rules`는 SKU 필수, 창고 선택, 안전재고 기준 수량, 활성 여부, 메모를 저장한다.
- SKU+창고 조합 중복은 서비스 검증으로 차단한다. 창고가 없는 rule은 전체 창고 기준으로 취급한다.
- 재고 부족 조회는 활성 rule만 대상으로 하며, 현재 수량이 기준 이하인 항목을 반환한다.
- 현재 수량은 `warehouse_location_stocks`의 SKU 위치 재고 합계를 우선 사용하고, 위치 재고가 없으면 기존 `warehouse_stocks`의 상품/창고 재고를 fallback으로 사용한다.

## API 엔드포인트

- `GET /api/admin/inventory-alert-rules`: 안전재고 기준 목록 조회. `INVENTORY_READ`
- `POST /api/admin/inventory-alert-rules`: 안전재고 기준 생성. `INVENTORY_WRITE`
- `PATCH /api/admin/inventory-alert-rules/{ruleId}`: 안전재고 기준 수정. `INVENTORY_WRITE`
- `PATCH /api/admin/inventory-alert-rules/{ruleId}/active`: 활성/비활성 변경. `INVENTORY_WRITE`
- `GET /api/admin/inventory-alerts/low-stock`: 재고 부족 항목 조회. `INVENTORY_READ`

## 남은 이슈

- 실제 외부 알림 채널 발송은 제외했다.
- 자동 발주, AI 수요 예측, 알림 이벤트 누적 테이블은 후속 고도화 범위다.
- 위치 재고가 0으로 등록된 경우 기존 창고 재고 fallback과 구분하는 정교한 source-of-truth 정책은 v0.5.8/v0.5.9 검증에서 후속 이슈로 정리한다.
