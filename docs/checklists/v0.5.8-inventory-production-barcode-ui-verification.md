## 검증 대상 화면

- `/admin/skus`: SKU 목록/검색/생성/수정/활성 토글/바코드 재발급
- `/admin/barcodes`: 바코드 검색/라벨 미리보기/출력 이력
- `/admin/barcode-stock`: 바코드 기반 재고 조회/입고/출고
- `/admin/production`: 생산 주문 생성/수정/시작/완료/취소/입고 이력
- `/admin/inventory`: 상품 재고 조회/입고/조정
- `/admin/stock-counts`: 재고 실사 세션/품목/완료 조정
- `/admin/warehouses`: 창고/창고별 재고/재고 이동
- `/admin/warehouse-locations`: 창고 위치/위치별 재고 조회
- `/admin/inventory-alerts`: 안전재고 기준/재고 부족 항목

## 핵심 흐름 수동 확인 후보

1. SKU 생성
2. 바코드 발급/재발급
3. 바코드 라벨 미리보기와 출력 이력 기록
4. 생산 주문 생성
5. 생산 완료 입고
6. 바코드 기반 재고 조회
7. 바코드 기반 입고/출고
8. 재고 실사 세션 생성과 실사 품목 저장
9. 실사 완료 시 차이 조정
10. 창고 위치 생성/수정/활성 상태 변경
11. 위치별 SKU 재고 조회
12. 안전재고 기준 생성/수정
13. 재고 부족 항목 조회
14. 관련 audit log 기록 확인

## 권한 검증 후보

- `SUPER_ADMIN`: 전체 재고/생산/바코드 화면과 변경 API 가능
- `ADMIN`: 운영 변경 권한 보유 범위에서 생성/수정 가능
- `MANAGER`: 조회 중심 화면 접근, 변경 버튼 비활성 또는 API 403
- 권한 없는 계정: 직접 URL 접근 시 관리자 보호 레이아웃의 권한 안내

## 남은 이슈

- 실제 브라우저 수동 확인은 로컬 서버 실행 상태에서 별도 수행이 필요하다.
- 실제 바코드 프린터 SDK, PDF 라벨 출력, 외부 알림 채널 발송은 v0.5 범위에서 제외됐다.
- 위치별 재고 0과 기존 상품/창고 재고 fallback의 정교한 source-of-truth 정책은 v0.5.9에서 후속 이관 후보로 정리한다.
