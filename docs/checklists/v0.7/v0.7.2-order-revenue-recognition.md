# v0.7.2 주문 매출 인식 체크리스트

작업 버전: `v0.7.2-order-revenue-recognition`

## 목표

결제/출고/배송 상태 기반 주문 매출 인식 기준을 구현한다.

## 구현 범위

- [x] 주문 단위 매출 인식 정책 구현
- [x] 결제 완료 시 `SALES` 회계 거래 생성
- [x] `referenceType=ORDER`, `referenceId=orderId`, `type=SALES` 기준 중복 생성 방지
- [x] 주문 매출 인식 수동 실행 API 추가
- [x] 주문 매출 인식 상태 조회 API 추가
- [x] 매출 인식 이벤트 목록 조회 API 추가
- [x] 회계 거래 생성 audit log 기록
- [x] 중복 매출 인식 skip audit log 기록
- [x] 프론트 accounting service에 매출 인식 API 타입 추가
- [x] 관리자 회계 화면 거래 안내 문구 갱신

## 정책

- 결제 상태가 `PAID`인 주문만 매출 인식 대상으로 본다.
- 주문 상태는 `PAID`, `PREPARING`, `SHIPPING`, `COMPLETED`만 허용한다.
- 동일 주문에 대한 `SALES` 거래가 이미 있으면 새 거래를 만들지 않고 기존 거래를 반환한다.
- 주문 취소와 환불 반제 처리는 v0.7.3에서 다룬다.

## 제외 범위

- 정산 배치.
- 회계 마감.
