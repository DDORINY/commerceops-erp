## 정책

- 정산 배치는 `DRAFT` 상태로 생성된다.
- 마감 API 호출 시 `CLOSED` 상태와 `closedAt`, `closedBy`를 기록한다.
- `CANCELLED` 상태 배치는 같은 기간 중복 생성 검사에서 제외한다.
- 정산 항목은 `accounting_transactions.occurredAt` 기준 기간 안에 있는 회계 거래를 대상으로 한다.
- 마감 후 금액 재계산과 회계 잠금은 후속 버전으로 이관한다.

## 제외 범위

- 실제 PG/택배사 정산 파일 수집.
- 정산 차액 자동 보정.
- 세금계산서/영수증 발행.
- 복식부기 마감 분개.
- 회계 기간 잠금과 수정 제한 전체 구현.
