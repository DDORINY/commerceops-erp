# v0.7.3 환불/반품 배송비 회계 처리 체크리스트

작업 버전: `v0.7.3-refund-return-fee-accounting`

## 목표

결제 환불, 반품 승인 환불, 반품 배송비 부담 주체를 회계 거래(`accounting_transactions`)로 반영한다.

## 정책

- 결제 환불 거래는 `referenceType=PAYMENT`, `type=REFUND` 기준으로 생성한다.
- 반품 환불 거래는 `referenceType=RETURN`, `type=REFUND` 기준으로 생성한다.
- 반품 배송비 거래는 `referenceType=RETURN`, `type=RETURN_FEE` 기준으로 생성한다.
- 반품 배송비가 없거나 부담 주체가 `UNDECIDED`이면 거래를 생성하지 않는다.
- 기존 단순 회계 내역(`accounting_entries`) 기록은 호환성을 위해 유지한다.

## 제외 범위

- 실제 PG 환불 정산 전문 연동.
- 환불 부분취소/부분반품 금액 계산.
- 복식부기 차변/대변 자동 분개.
- 정산 배치 마감 반영.
- 반품 배송비 청구/차감 결제 자동화.
