작업 브랜치: `v0.6.1-outbound-order-model`

## API

- `GET /api/admin/outbound-orders`: 출고 지시 목록 조회, `OUTBOUND_READ`
- `GET /api/admin/outbound-orders/{outboundOrderId}`: 출고 지시 상세 조회, `OUTBOUND_READ`
- `POST /api/admin/outbound-orders`: 주문 기준 출고 지시 생성, `OUTBOUND_MANAGE`
- `PATCH /api/admin/outbound-orders/{outboundOrderId}`: 출고 창고/메모 수정, `OUTBOUND_MANAGE`
- `PATCH /api/admin/outbound-orders/{outboundOrderId}/pick`: 피킹 완료 처리, `OUTBOUND_MANAGE`
- `PATCH /api/admin/outbound-orders/{outboundOrderId}/cancel`: 출고 지시 취소, `OUTBOUND_MANAGE`

## 정책

- 출고 지시는 주문과 창고를 기준으로 생성한다.
- 출고 품목은 주문 품목을 기준으로 생성하며, 상품에 연결된 최신 SKU가 있으면 함께 연결한다.
- 기존 주문 중 SKU가 없는 상품도 출고 지시를 만들 수 있도록 `sku_id`는 nullable로 둔다.
- `pick` 처리는 v0.6.1 범위에서 전체 품목의 `pickedQuantity`를 주문 수량과 동일하게 맞추고 상태를 `PICKED`로 변경한다.
- 실제 바코드 스캔 검수, 재고 차감, 송장번호 생성, 배송 완료 처리는 후속 v0.6.x로 이관한다.
