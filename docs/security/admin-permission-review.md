# 관리자 권한 검토

## 이번 긴급 패치 범위

역할 변경, 권한 그룹 및 사용자 권한 그룹 변경, HR 직원 관리와 회계 쓰기 작업은 Spring Security URL 규칙에서 `ADMIN` 또는 `SUPER_ADMIN`만 허용하도록 제한한다. 프런트 메뉴 숨김은 보안 경계로 사용하지 않는다.

기존 `PermissionChecker`는 권한 그룹에 의해 MANAGER가 권한을 가질 수 있는 구조이므로, 역할/권한 관리 같은 보안 민감 경로는 URL 계층에서 추가로 ADMIN-only 제한을 둔다.

## 남은 구조 개선

- 권한 코드를 역할별로 재분류하고 Controller의 `@PreAuthorize`로 이중 강제
- 회계 읽기와 쓰기 권한을 분리
- 감사 로그는 삭제/수정 API를 제공하지 않고 append-only 저장소로 운영
- 권한 변경 시 대상 사용자와 행위자의 역할 변경 감사 이벤트를 추가
