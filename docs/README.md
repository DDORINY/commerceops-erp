# CommerceOps ERP 문서 안내

이 폴더는 CommerceOps ERP의 제품 범위, 구현 현황, 기술 명세와 운영 기준을 관리하는 단일 진입점이다.

## 문서 읽는 순서

1. [현재 구현 현황](./CURRENT_STATE.md) — 코드 기준으로 지금 가능한 것과 막힌 것
2. [제품 로드맵](./PRODUCT_ROADMAP.md) — 제품이 어느 방향으로 확장되는지
3. [버전별 구현 계획](./VERSION_PLAN.md) — 버전마다 어디서 시작해 어디까지 끝낼지와 체크리스트
4. [서비스 확장 계획](./SERVICE_EXPANSION.md) — Claude 로드맵 이후 운영 서비스와 SaaS까지의 확장
5. [문서화 운영 기준](./DOCUMENTATION_GUIDE.md) — 무엇을 언제 갱신해야 하는지

## 상세 기술 문서

| 문서 | 역할 | 갱신 시점 |
| --- | --- | --- |
| [API 명세](./API.md) | 요청·응답·권한·오류 계약 | API 추가/변경 시 |
| [DB 스키마](./DB_SCHEMA.md) | 테이블·관계·제약·데이터 정책 | 엔티티/마이그레이션 변경 시 |
| [백엔드 구조](./BACKEND_STRUCTURE.md) | 패키지·계층·트랜잭션 구조 | 백엔드 구조 변경 시 |
| [프론트엔드 구조](./FRONTEND_STRUCTURE.md) | 라우트·컴포넌트·상태 관리 | 화면/구조 변경 시 |

기존 상세 기술 문서는 초기 설계안 성격의 내용도 포함한다. 현재 사실 여부는 항상 `CURRENT_STATE.md`와 실제 코드로 먼저 확인하고, 구현하면서 상세 문서를 동기화한다.

## 상태 표기

| 표기 | 의미 |
| --- | --- |
| ✅ 완료 | 구현·테스트·문서가 완료 기준을 통과함 |
| 🟡 구현 중 | 코드 일부 또는 화면이 있으나 완료 기준을 통과하지 못함 |
| 🔴 차단 | 빌드 실패 등 다음 작업보다 먼저 해결할 문제 |
| ⬜ 예정 | 아직 시작하지 않음 |

`파일이 존재함`은 `완료`가 아니다. 완료 여부는 [버전별 구현 계획](./VERSION_PLAN.md)의 공통 완료 조건으로 판정한다.
