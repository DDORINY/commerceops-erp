# 프로젝트 개요

## 기본 정보

- 프로젝트명: CommerceOps ERP
- 한 줄 설명: 쇼핑몰 기능과 ERP 운영 기능을 결합한 AI 커머스 ERP 포트폴리오 프로젝트.
- 기준 버전: `v1.0.1`

## 프로젝트 목적

CommerceOps ERP는 상품 판매 화면에 머무르지 않고 상품, 주문, 결제, 재고, 생산, 배송, 회계, 권한, 감사 로그, AI 운영 화면을 하나의 운영 흐름으로 연결한다. 실제 운영 데이터 대신 합성 샘플 데이터와 baseline 모델을 사용해 AI 수요 예측, 이상 탐지, 리뷰 분석 구조까지 포트폴리오에서 재현 가능하게 정리한다.

## 해결하려는 문제

- 쇼핑몰 주문 이후 재고, 출고, 배송, 반품, 정산 데이터가 분리되어 운영 흐름을 추적하기 어렵다.
- 관리자 권한이 단순 role에만 의존하면 실제 업무별 권한 제어가 어렵다.
- 재고/바코드/출고/배송 데이터가 AI 학습 데이터로 정리되지 않으면 운영 자동화로 확장하기 어렵다.

## 주요 사용자

- 일반 구매자
- 쇼핑몰 운영자
- 상품/전시 담당자
- 재고/창고 담당자
- 배송/CS 담당자
- 최고관리자 및 권한 관리자

## 핵심 운영 흐름

```text
상품 등록 -> SKU/바코드 -> 재고/생산 입고 -> 주문/결제
  -> 출고 지시 -> 바코드 검수 -> 송장/배송 -> 반품/환불
  -> 감사 로그/권한 통제 -> 회계/정산 -> AI 데이터셋/AI 운영
```

## 현재 구현된 주요 모듈

- 인증/세션/JWT
- 상품/카테고리/배너 CMS
- 주문/결제/환불
- 리뷰/문의/알림
- 직원/권한/감사 로그
- 사업자/약관 설정
- SKU/바코드/생산 입고
- 재고 실사/창고 위치/안전재고
- 출고 지시/택배사/송장/배송 추적/반품 배송
- 회계 원장/정산/기간 마감/회계 리포트
- AI 데이터셋/export/마스킹/학습/평가/운영 화면

## AI 데모 기준

- 실제 고객 데이터는 사용하지 않는다.
- 개인정보가 없는 합성 샘플 데이터와 baseline `.pt` 모델을 사용한다.
- 관리자 AI 화면은 자동 의사결정이 아니라 운영자가 검토할 참고 지표로 제공한다.
