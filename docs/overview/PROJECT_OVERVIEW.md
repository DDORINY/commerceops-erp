# 프로젝트 개요

## 기본 정보

- 프로젝트명: CommerceOps ERP
- 한 줄 설명: 쇼핑몰 기능과 ERP 운영 기능을 결합한 AI 커머스 ERP 포트폴리오 프로젝트.
- 기준 버전: `v0.6.9`

## 프로젝트 목적

CommerceOps ERP는 상품 판매 화면에 머무르지 않고 상품, 주문, 결제, 재고, 생산, 배송, 권한, 감사 로그를 하나의 운영 흐름으로 연결한다. 이후 v0.8/v0.9에서 AI 수요 예측, 이상 탐지, 리뷰 분석으로 확장해 실무 ERP의 핵심 흐름을 축소 구현한 포트폴리오를 완성하는 것이 목표다.

## 해결하려는 문제

- 쇼핑몰 주문 이후 재고, 출고, 배송, 반품, 정산 데이터가 분리되어 운영 흐름을 추적하기 어렵다.
- 관리자 권한이 단순 role에만 의존하면 실제 업무별 권한 제어가 어렵다.
- 재고/바코드/출고/배송 데이터가 AI 학습 데이터로 정리되지 않으면 운영 자동화로 확장하기 어렵다.

## 주요 사용자

- 일반 구매자
- 쇼핑몰 운영자
- 상품/전시 담당자
- 재고/창고 담당자
- 배송/CS 담당자
- 최고관리자 및 권한 관리자

## 핵심 운영 흐름

```text
상품 등록 -> SKU/바코드 -> 재고/생산 입고 -> 주문/결제
  -> 출고 지시 -> 바코드 검수 -> 송장/배송 -> 반품/환불
  -> 감사 로그/권한 통제 -> 회계/정산 후보 -> AI 데이터셋 후보
```

## 현재 구현된 주요 모듈

- 인증/세션/JWT
- 상품/카테고리/배너 CMS
- 주문/결제/환불
- 리뷰/문의/알림
- 직원/권한/감사 로그
- 사업자/약관 설정
- SKU/바코드/생산 입고
- 재고 실사/창고 위치/안전재고
- 출고 지시/택배사/송장/배송 추적/반품 배송

## 향후 AI 확장 방향

- v0.8: synthetic ERP dataset, 전처리, 학습, 평가, 모델 산출 구조.
- v0.9: 관리자 화면에서 AI 추천, 수요 예측, 이상 탐지, 리뷰 분석을 운영 기능으로 연결.
