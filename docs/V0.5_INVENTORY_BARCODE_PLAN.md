# v0.5 재고관리/생산 입고/바코드 자동화 계획 후보

기준 버전: `v0.4.9-hr-permission-release-verification`

문서 상태: v0.4.9 검증에서 정리한 v0.5 이관 후보 문서다. v0.5.0에서 범위, 제외 범위, DB/API/UI 설계를 다시 확정한다.

## 목표

v0.5에서는 상품, 옵션, 주문, 창고 기반 위에 실무 ERP의 재고 운영 흐름을 축소 구현한다. 생산 입고, 바코드 생성, 라벨 출력, 재고 실사, 창고 위치 관리, 안전재고 알림을 중심으로 재고 운영 기반을 고도화한다.

## 핵심 요구사항 후보

- 상품을 직접 생산하는 경우 생산 입고가 가능해야 한다.
- 생산 완료 시 재고가 자동 증가해야 한다.
- SKU 또는 옵션 단위 바코드를 자동 생성할 수 있어야 한다.
- 바코드 라벨 출력 흐름을 제공해야 한다.
- 바코드 기반 재고 조회가 가능해야 한다.
- 입고, 출고, 조정, 이동 이력을 기록해야 한다.
- 안전재고 기준과 재고 부족 알림 후보를 설계해야 한다.
- v0.6 유통관리, 송장, 출고 자동화와 연결 가능한 구조로 설계해야 한다.

## v0.4에서 넘어온 주의사항

- v0.4.9 기준으로 인사/권한/감사 로그/사업자 설정 저장은 완료 상태다.
- v0.5 구현 API는 기존 role 1차 접근과 permission code 세부 실행 정책을 따라야 한다.
- 재고 변경, 생산 입고, 바코드 생성/출력, 실사 조정은 감사 로그 후보로 기록해야 한다.
- 관리자 사이드바는 permission 기반 노출 정책을 유지해야 한다.
