# AI 운영 화면 포트폴리오 데모 시나리오

## 목적

v0.9 AI 운영 기능을 발표나 면접에서 설명할 때 사용할 시연 순서와 핵심 설명 포인트를 정리한다.

CommerceOps ERP의 AI 기능은 실제 운영 고객 데이터를 수집하지 않는 포트폴리오 환경을 전제로 한다. 따라서 개인정보가 없는 합성 샘플 데이터와 baseline `.pt` 모델, rule-based 운영 지표를 조합해 “AI 커머스 ERP가 어떤 운영 흐름을 가질 수 있는지”를 보여주는 데 초점을 둔다.

## 데모 전 준비

### 1. 백엔드 실행

```powershell
cd backend
.\gradlew.bat bootRun
```

### 2. 프론트엔드 실행

```powershell
cd frontend
npm.cmd run dev
```

### 3. AI 데모 모델 생성 확인

이미 `.pt` 모델을 만든 경우 아래 파일이 로컬에 존재한다.

```text
ai/models/checkpoints/commerceops_demo_demand_baseline.pt
ai/models/checkpoints/commerceops_demo_review_sentiment_baseline.pt
```

모델 파일은 Git 추적 대상이 아니다. 재현이 필요하면 아래 명령으로 다시 만들 수 있다.

```powershell
python ai/scripts/train_baseline.py --config ai/configs/train_demo_demand.json
python ai/scripts/train_baseline.py --config ai/configs/train_demo_review.json
```

## 관리자 AI 화면 시연 순서

### 1. AI 운영 개요

경로: `/admin/ai`

설명 포인트:

- v0.8에서 데이터셋 export, 개인정보 마스킹, baseline 학습/평가 파이프라인을 만들었다.
- v0.9에서는 그 결과를 관리자 운영 화면에 연결했다.
- AI 결과는 자동 처리 기능이 아니라 관리자 의사결정 보조 지표다.

### 2. AI 상품 추천

경로: `/admin/ai/recommendations`

설명 포인트:

- 상품 노출 상태, 판매 상태, 재고, 태그, 검색 키워드, 이미지 여부를 바탕으로 추천 후보 점수를 계산한다.
- 실제 개인화 추천이나 자동 전시 반영은 하지 않는다.
- 포트폴리오에서는 “추천 후보를 운영자가 검토하는 흐름”을 보여준다.

### 3. AI 수요 예측

경로: `/admin/ai/demand-forecast`

설명 포인트:

- 상품별 현재 재고, 안전재고, 데모 수요지수를 기준으로 재고 소진 위험을 표시한다.
- 실제 서비스라면 주문 시계열, 시즌성, 프로모션, 외부 이벤트를 추가해야 한다.
- 현재 버전에서는 자동 발주나 생산 지시 없이 관리자 보충 판단용으로만 표시한다.

### 4. AI 리뷰 분석

경로: `/admin/ai/review-analysis`

설명 포인트:

- 리뷰 평점과 마스킹된 본문을 기준으로 긍정/중립/부정 확인 후보를 표시한다.
- 리뷰 본문은 개인정보 마스킹 유틸을 거친다.
- 자동 숨김이나 제재가 아니라 운영자가 확인할 리뷰를 찾는 보조 화면이다.

### 5. AI 이상 주문 탐지

경로: `/admin/ai/order-anomalies`

설명 포인트:

- 주문 금액, 할인율, 주문/결제 상태 불일치 후보를 기준으로 이상 주문 후보를 표시한다.
- 자동 주문 차단은 하지 않는다.
- 운영자는 주문 관리 화면에서 실제 주문 상태와 결제 상태를 추가 확인한다.

### 6. AI 리스크 알림

경로: `/admin/ai/risk-alerts`

설명 포인트:

- 재고 부족 리스크와 정산 확인 필요 리스크를 한 화면에서 보여준다.
- 재고 리스크는 현재 재고와 안전재고를 기준으로 계산한다.
- 정산 리스크는 정산 상태와 순정산금액을 기준으로 계산한다.
- 자동 입고나 정산 마감은 하지 않는다.

### 7. AI 리포트

경로: `/admin/ai/reports`

설명 포인트:

- 각 AI 운영 화면의 모델명, 근거 데이터, 해석 기준, 한계를 카드로 설명한다.
- 포트폴리오 검토자가 “왜 이 점수가 나왔는지”를 이해할 수 있게 만든 화면이다.
- SHAP/LIME, LLM 요약, MLOps 플랫폼은 제외 범위로 남겨두었다.

## 발표용 핵심 문장

> 이 프로젝트의 AI 기능은 실제 고객 데이터를 쓰지 않고, 합성 샘플 데이터와 마스킹된 export 구조를 기반으로 학습/평가/운영 화면 연결 흐름을 재현했습니다. 자동 의사결정보다는 ERP 운영자가 검토할 수 있는 추천, 예측, 분석, 탐지 후보와 근거 설명을 제공하는 구조입니다.

## 의도적으로 제외한 것

- 실제 운영 고객 데이터 수집
- 외부 LLM API 연동
- 자동 주문 차단
- 자동 발주/입고
- 자동 정산 마감
- S3, SageMaker, MLflow 같은 완전한 MLOps 구성
- 실제 서비스 수준의 개인화 추천 모델

## 면접 질문 대응 포인트

### 실제 데이터가 없는데 왜 AI 기능을 만들었나?

포트폴리오 프로젝트라 실제 고객 데이터를 수집할 수 없기 때문에 개인정보 없는 합성 데이터로 재현 가능한 파이프라인을 만들었다. 중요한 것은 운영 데이터가 어떤 형태로 AI 학습/평가/화면 연결까지 이어지는지 설계한 점이다.

### `.pt` 모델은 실제로 만들어지나?

PyTorch가 설치된 환경에서는 `train_baseline.py`가 `.pt` checkpoint를 생성한다. 현재 프로젝트에서는 수요 예측과 리뷰 감성 baseline 모델 생성 흐름을 제공한다.

### 운영 화면의 AI 점수는 실제 모델 추론인가?

v0.9 관리자 화면은 포트폴리오 데모 안정성을 위해 rule-based 점수와 baseline 모델 설명을 함께 사용한다. 실제 추론 서버 분리, 모델 로딩, 실시간 예측은 후속 고도화 범위다.

### 개인정보는 어떻게 다루나?

AI dataset export에는 개인정보 마스킹 유틸을 적용했고, 리뷰 분석 화면에서도 마스킹된 본문을 표시한다. 실제 운영에서는 별도 개인정보 처리 정책, 보관 기간, 접근 로그가 추가로 필요하다.
