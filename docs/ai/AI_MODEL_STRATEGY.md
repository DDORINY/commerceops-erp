# AI 모델 전략

## 기본 방향

CommerceOps ERP의 핵심 AI는 LLM이 아니라 운영 데이터 분석형 AI다. LLM은 관리자 보조 기능으로 제한하고, 주요 기능은 수요 예측, 이상 탐지, 리뷰 분석, 추천 후보로 구성한다.

## 우선 모델 후보

### 수요 예측

- SKU별 판매량과 재고 흐름을 기반으로 단기 수요를 예측한다.
- v0.8에서 synthetic dataset과 baseline 모델을 만든다.

### 이상 주문/재고 탐지

- 주문 금액, 반복 취소/환불, 급격한 재고 변동을 탐지한다.
- rule-based baseline에서 시작하고 Isolation Forest/AutoEncoder를 후보로 둔다.

### 리뷰 감성 분석

- 리뷰 텍스트를 긍정/부정/중립 또는 이슈 카테고리로 분류한다.
- 초기에는 TF-IDF + Logistic Regression을 우선한다.

### 상품 추천 후보

- 인기 상품, 같이 구매한 상품, 카테고리 기반 추천을 후보로 둔다.
- 대형 추천 시스템은 v0.9 이후 고도화 후보로 둔다.

## LLM 사용 기준

- 핵심 예측 모델로 사용하지 않는다.
- 리뷰 답변 초안, 운영 리포트 요약, 이상 탐지 설명 같은 보조 기능으로 제한한다.
- 사용량 제한, 캐싱, 비용 추정 로그를 둔다.

## 제외 범위

- 객체 탐지.
- 영상 분석.
- 대형 LLM 직접 호스팅.
- AWS 1대 서버에서 대규모 GPU 학습.
